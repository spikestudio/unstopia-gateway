# セルフレビュー反復ループ（共通参照）

各スキルがセルフレビューを実行する際の反復ループ手順。レビュー品質基準は `aidd-framework/references/review-standards.md` を参照する。

## 反復ループ手順

```
レビュー実施 → MUST FIX あり → 修正 → 再レビュー（最大3回）
    ↑                                         |
    +---- MUST FIX が残っている（3回未満）-----+
```

### ラウンド 1: 初回レビュー

1. スキル固有の観点 + `review-standards.md` の品質基準でレビューを実施する
2. 指摘を `review-standards.md` のフォーマットで一覧化する
3. **MUST FIX が 0 件** → PASS。ブリーフィングへ進む
4. **MUST FIX が 1 件以上** → 修正してラウンド 2 へ

### ラウンド 2〜3: 再レビュー

1. 前ラウンドの MUST FIX を全て修正する
2. **修正箇所のみでなく、全観点で再レビューする**（修正が新たな問題を生む可能性があるため）
3. **MUST FIX が 0 件** → PASS。ブリーフィングへ進む
4. **MUST FIX が残っている + ラウンド 3 未満** → 修正して次ラウンドへ

### 3 回で解消しない場合

3 ラウンド実施しても MUST FIX が残る場合:

1. 残存する MUST FIX を一覧でユーザーに報告する
2. 各指摘について「なぜ 3 回の修正で解消できなかったか」の理由を添える
3. ユーザーの判断を仰ぐ（追加修正指示 / 許容判断 / 方針変更）

## SHOULD FIX / CONSIDER の扱い

- SHOULD FIX / CONSIDER は反復ループの対象外（MUST FIX のみが反復条件）
- ブリーフィングで SHOULD FIX / CONSIDER の一覧を提示し、対応要否をユーザーに判断いただく

## レビュー結果の報告フォーマット

各ラウンドの結果を以下の形式で記録する:

```
### セルフレビュー結果（ラウンド N/3）

| # | 重大度 | 観点 | 指摘内容 | 対応状況 |
|---|--------|------|---------|---------|
| 1 | MUST FIX | [観点] | [指摘の要約] | [修正済み / 未修正] |
| 2 | SHOULD FIX | [観点] | [指摘の要約] | — |

**判定:** PASS / FAIL（MUST FIX 残 N 件）
```

## 使い方

各スキルのセルフレビュー Step で以下のように参照する:

```markdown
## Step N: セルフレビュー

セルフレビュー反復ループを実施する（aidd-framework/references/self-review-loop.md 参照）。
レビュー品質基準は aidd-framework/references/review-standards.md に従う。
スキル固有の追加観点:

- [スキル固有の観点をここに記載]
```
