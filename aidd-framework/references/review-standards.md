# レビュー品質基準（共通参照）

各スキルがセルフレビューを実行する際の共通品質基準。重大度判定・指摘フォーマット・自己監査・密度チェックの基準を定義する。

## 重大度判定ルール

| 判断軸 | 重大度 |
|--------|--------|
| 規約違反 | 🔴 MUST FIX（例外なし） |
| AC 不準拠 | 🔴 MUST FIX |
| 設計整合性の不一致 | 🔴 MUST FIX |
| テンプレート必須セクション欠落 | 🔴 MUST FIX |
| 上位ドキュメントとの矛盾 | 🔴 MUST FIX |
| 曖昧な記述 | 🔴 MUST FIX |
| パフォーマンス改善（規約外） | 🟡 SHOULD FIX |
| 可読性・保守性の改善 | 🟡 SHOULD FIX / 🟢 CONSIDER |

## 指摘フォーマット

```
### [重大度] [タイトル]
- **ファイル:** [ファイルパス:行番号]
- **根拠:** [どの規約・AC・設計ドキュメントの何に違反しているか]
- **問題:** [何が問題か]
- **修正指示:** [具体的にどう修正すべきか]
```

## 重大度の自己監査

MUST FIX 以外の全指摘に対して以下を確認し、該当すれば 🔴 に格上げする:

1. **規約突合**: `docs/conventions/` または CLAUDE.md の規約に違反していないか
2. **AC 突合**: Task 定義の完了条件に関わっていないか
3. **設計突合**: 設計成果物との不整合ではないか

SHOULD FIX / CONSIDER にした理由を「〜の規約に該当しないため」と明記する。

## レビュー密度チェック

以下のいずれかに該当する場合、レビューが不十分な可能性がある:

| 条件 | 警告 |
|------|------|
| 変更行数 100+ で指摘 0 件 | レビューが十分か確認 |
| 変更ファイル 5+ で全て CONSIDER のみ | 規約・AC の突合が十分か確認 |
| AC エビデンス表にテストエビデンスが 0 件 | テストの存在確認ができていない |
| 規約ドキュメントへの具体的参照が 0 件 | 規約との突合が不十分 |

## 使い方

各スキルのセルフレビュー（例: /aidd-impl の selfcheck.md、/aidd-epic-review の 3 観点チェック）で、この品質基準を参照して重大度判定・指摘フォーマットを統一する。
