# マルチエージェント実行ガイド

## 概要

AIDD-FW のオーケストレータースキルは、Claude Code の `subagent_type` パラメータを使って専門化されたペルソナエージェントを並行起動できる。このガイドでは、マルチエージェント実行の仕組み・制約・ベストプラクティスを記述する。

## subagent_type の動作仕様

### 基本構文

```
Agent(subagent_type="aidd-analyst", prompt="...")
```

- `subagent_type` に指定する値はエージェント定義ファイルのファイル名（拡張子なし）
- Claude Code がエージェント定義ファイルを読み込み、そのペルソナとして動作する
- 1 メッセージ内に複数の Agent 呼び出しを記述すると並行起動される

### ペルソナエージェント一覧

| ペルソナ | ファイル | 専門性 | 判断ベース |
|---------|--------|--------|-----------|
| aidd-analyst | `agents/aidd-analyst.md`（プラグイン） | 業務・ドメイン分析 | Phase 定義書、Epic 仕様書、プロジェクト憲章、用語集 |
| aidd-architect | `agents/aidd-architect.md`（プラグイン） | 設計判断・トレードオフ評価 | ADR、技術スタック、アーキテクチャ概要、規約 |
| aidd-builder | `agents/aidd-builder.md`（プラグイン） | 実装・構築 | 規約、設計成果物、技術スタック、既存コードパターン |

### 配置方法

Claude Code はインストール済みプラグインの `agents/` ディレクトリを自動認識する（優先度 5）。プロジェクトへのコピーは不要。プロジェクト独自のエージェントを追加したい場合は `.claude/agents/` に配置する（優先度 3 でプラグインより優先される）。

## 制約事項

### コンテキスト共有

- 各ペルソナエージェントは**独立したコンテキスト**で動作する。親のオーケストレーターのコンテキストは共有されない
- ペルソナが参照すべき情報は `prompt` パラメータ内にファイルパスとして渡し、ペルソナ側で Read する
- オーケストレーターの分析結果をペルソナに渡す場合、`prompt` に直接テキストとして埋め込む

### 同時起動

- 1 メッセージ内の複数 Agent 呼び出しは並行実行される
- 実用的な同時起動数は **3〜5 程度**。それ以上はレスポンス品質が低下する可能性がある
- 各ペルソナの応答時間は独立。最も遅いペルソナの完了を待ってからオーケストレーターが集約を開始する

### model 指定

- ペルソナの frontmatter `model: sonnet` は、そのペルソナの実行モデルを指定する
- オーケストレーター自身のモデルとは独立して動作する
- コスト・速度のバランスで `sonnet` を推奨。`opus` を指定するとより深い分析が得られるが応答時間が増加する

### ファイル操作

- **ペルソナは読み取り専門**（Read, Grep, Glob）で運用する
- 書き込み（Edit, Write）はオーケストレーターが結果を集約した後に実行する
- これにより並行実行時のファイル書き込み競合を回避する

## いつ並行実行すべきか（判断基準）

| 状況 | 推奨 | 理由 |
|------|------|------|
| 複数の独立した観点で分析が必要 | **並行実行** | 例: decision-facilitator の 3 ペルソナ分析 |
| 前の結果が次の入力に必要 | **逐次実行** | 例: mock の recommend → customize → verify |
| 単一の専門性で十分 | **単一ペルソナ** | 例: screen-design の builder によるドラフト生成 |
| 分析不要、機械的な処理 | **セルフレビューループ** | 例: 各スキル内蔵のセルフレビューループ（検証→修正→再検証） |

## エラー時のフォールバック

### subagent_type 非対応環境

`subagent_type` パラメータが利用できない場合、プレーンな Agent 呼び出しで代替する:

```
# フォールバック
Agent(prompt="あなたは aidd-analyst（業務・ドメイン分析の専門家）です。以下を分析してください。...")
```

### ペルソナ未認識

プラグインが正しくインストールされていない場合も同様にフォールバックする。`/plugin install aidd-fw` でプラグインを再インストールすれば認識される。

### タイムアウト

1 つのペルソナが長時間応答しない場合、オーケストレーターは完了済みの結果で集約を進め、未完了のペルソナについてユーザーに報告する。

## オーケストレーター設計パターン

### 並行分析パターン（decision-facilitator）

```
# 1 メッセージで 3 ペルソナを並行起動
Agent(subagent_type="aidd-analyst", prompt="業務視点で分析...")
Agent(subagent_type="aidd-architect", prompt="設計視点で分析...")
Agent(subagent_type="aidd-builder", prompt="実装視点で分析...")

# オーケストレーターが結果を集約して推奨案を生成
```

### 逐次専門家パターン（screen-design）

```
# Step 1: analyst がモックを解析
Agent(subagent_type="aidd-analyst", prompt="モック解析...")

# Step 2: builder が解析結果を元にドラフト生成
Agent(subagent_type="aidd-builder", prompt="仕様書ドラフト生成...")
```

### インライン処理 + ペルソナ混合パターン（mock）

```
# インライン: テンプレート推薦（catalog.json マッチング）
# → スキル内で直接実行

# ペルソナ: カバレッジ検証
Agent(subagent_type="aidd-builder", prompt="カバレッジ検証...")
```
