# スキル開発ガイド

新しいスキルを作成・修正する際の品質基準。

## ファイル構造

```
skills/
  aidd-<name>/
    SKILL.md      ← 必須。スキル定義本体
```

## SKILL.md の必須要素

### frontmatter

```yaml
---
name: aidd-<name>
description: 1行で用途を説明。argument-hint がある場合は引数の形式も記載
argument-hint: "[引数の形式]"   # 引数を取る場合のみ
---
```

- `name` と `description` は必須
- `argument-hint` は引数を取るスキルのみ

### 「重要なルール」セクション

全スキルに以下を含めること:

```markdown
## 重要なルール

- 日本語で対話する
- **FRAMEWORK.md の意思決定基準に従う** — 優先順序 1〜5 に該当する問題の見送り・後回しを推奨しない
- [スキル固有のルール]
- コミットはユーザーの判断に委ねる（ファイル変更を伴うスキルのみ）
```

## 引数ハンドリングの統一パターン

引数を取るスキルは Step 1 で以下の 3 パターンを全て定義すること:

### パターン 1: 引数あり

```markdown
**引数あり（例: `/aidd-xxx G6 ES-003`）:**

- 引数から対象を特定する
```

### パターン 2: 引数なし（自動推定）

```markdown
**引数なし:**

- GitHub Issues（`gh issue list`）からプロジェクト状態を分析
- 推定結果をユーザーに確認してから実行
```

### パターン 3: 推定失敗時（フォールバック）

コードブロック形式でエラーメッセージを表示し、引数の指定方法を案内する:

````markdown
**推定できない場合:**

```
対象を自動推定できませんでした。以下のいずれかを指定してください:
- `/aidd-xxx G6 ES-003` — [説明]
- `/aidd-xxx <名前>` — [説明]
```
````

**ユーザーに「何を指定すればよいか分からない」状態で丸投げしない。** 具体的な引数例を必ず含める。

## 完了メッセージの統一パターン

スキルの最終ステップには、次のアクションへの導線を含めること:

```markdown
次のステップ:
→ `/aidd-next` で次のアクションを確認
```

- パイプライン上のスキルは、次に実行すべきスキルを `→` 形式で案内する
- 完了メッセージには `/aidd-next` への導線を含める

## パイプラインへの組み込み

新しいスキルをパイプラインに追加する場合、以下の全てを更新すること:

| 更新対象 | ファイル | 内容 |
|---------|--------|------|
| ゲート×スキル対応表 | `aidd-framework/FRAMEWORK.md` | 新スキルの行を追加 |
| スキルパイプライン図 | `aidd-framework/templates/claude-md.md` | フロー図に追加 |
| ゲートチェックリスト | `aidd-framework/checklists/g{N}-*.md` | 前提条件・チェック項目を追加 |
| doctor の検査対象 | `skills/aidd-doctor/SKILL.md` | パイプラインスキル一覧に追加 |

## チェックリスト

新スキル作成時のセルフチェック:

- [ ] frontmatter に name, description が定義されている
- [ ] 「重要なルール」に意思決定基準・日本語対話が含まれている
- [ ] 引数を取る場合、3 パターン（あり / なし / 推定失敗）が定義されている
- [ ] 完了メッセージに次のアクションへの導線がある
- [ ] パイプライン上のスキルなら、関連ドキュメント 4 箇所が更新されている
- [ ] ユーザー確認なしに自動実行するステップがない
