## レビューの判断軸（優先順序）

以下の優先順序で判断すること。上位の判断軸を下位の理由で覆してはならない。

1. **規約準拠** — `docs/conventions/` および CLAUDE.md の規約に違反していないか
2. **AC 準拠** — Task 定義の完了条件・Epic 仕様書の AC を満たしているか
3. **設計整合性** — ADR・設計成果物との整合が取れているか
4. **品質・最適化** — パフォーマンス、可読性、保守性の改善提案

## 必ずチェックする項目

<!-- プロジェクト固有の規約チェック項目を以下に追加する -->

### 規約準拠（違反は必ず指摘）

- [ ] 命名規則に従っているか（`docs/conventions/` の命名規則を参照）
- [ ] ディレクトリ構造・ファイル配置ルールに従っているか
- [ ] レイヤー間の依存方向が正しいか（上位→下位のみ）
- [ ] エラーハンドリング方針に従っているか
- [ ] テスト規約に従っているか（命名、構造、カバレッジ基準）
- [ ] CLAUDE.md の「絶対禁止」事項に違反していないか
- [ ] Git 運用ルール（ブランチ戦略、コミット規約）に従っているか

### AC 準拠

- [ ] Task 定義の完了条件が全て実装されているか
- [ ] 完了条件にない機能が実装されていないか（過剰実装）
- [ ] テストが AC に基づいて書かれているか（実装のテストになっていないか）

### AI 生成コード特有のリスク

- [ ] 存在しない API・ライブラリ・設定を参照していないか（ハルシネーション）
- [ ] Task のスコープ外の変更が含まれていないか（過剰実装）
- [ ] 規約と異なるパターンが使われていないか（学習データへの引きずられ）
- [ ] 入力値検証・認可チェックが漏れていないか

## 重大度の定義

| 重大度 | 定義 |
|--------|------|
| 🔴 **Normal** | 規約違反・AC 不準拠・セキュリティ問題。マージ前に修正必須 |
| 🟡 **Nit** | 改善推奨だが必須ではない |

**規約違反は必ず Normal（修正必須）として報告すること。** 「実害がない」「コストが高い」は免除理由にならない。

<!-- レビュー指摘: /aidd-review epicと重大度用語が異なり混乱する可能性。対応関係を注記 -->
> **`/aidd-review epic` との対応:** より厳密なレビューが必要な場合は `/aidd-review epic` スキルを使用すること。重大度の対応関係: Normal = MUST FIX、Nit = SHOULD FIX / CONSIDER。
