## Section 2: 技術基盤定義（旧 G0.1）

### 技術スタック・アーキテクチャ

- [ ] 技術スタックの全選定に ADR が存在する
- [ ] アーキテクチャ方針が ADR で定義されている
- [ ] 技術スタックマスタドキュメントが作成されている
- [ ] アーキテクチャ概要マスタドキュメントが作成されている

### 規約ドキュメント群（必須整備項目）

各規約は「AI がこの規約だけを読んで、規約に準拠したコードを生成できるか」を基準に品質を判断する。具体例（正例・誤例）が各規約に最低1つ含まれていること。

- [ ] **命名規則** — 変数、関数、ファイル、ディレクトリの命名規則が定義されている
- [ ] **ディレクトリ構造** — プロジェクトのディレクトリ構成とファイル配置ルールが定義されている
- [ ] **レイヤー間のルール** — 依存方向、呼び出しパターン、データの受け渡し方が定義されている
- [ ] **エラーハンドリング方針** — エラーの伝播方法、ログ出力、リカバリ方針が定義されている
- [ ] **テスト規約** — テストの命名、構造、どこまでテストするかが定義されている
- [ ] **禁止事項** — やってはいけないことの明示的なリストが存在する（最低5項目以上）
- [ ] **Git 運用ルール** — ブランチ戦略、マージ方式、コミットメッセージ規約が定義されている

### AI ツール設定

- [ ] AI ツール設定ファイル（CLAUDE.md 等）が作成され、規約ドキュメント群へのポインタが記載されている

### CI/CD・開発環境

- [ ] CI パイプラインが動作し、リント・テスト・ビルドが自動実行される
- [ ] ローカル CI フック（pre-commit / pre-push）が設定され、CI と同等のチェックがローカルで事前実行される
- [ ] CI が失敗した場合、PR のマージがブロックされる設定になっている
- [ ] 開発環境が1コマンドで起動できる（セットアップ手順がドキュメント化されている）
- [ ] 前提条件（ランタイム・ツールのバージョン）が README に明記されている
- [ ] 初回セットアップ後にテストスイートが通過する
- [ ] デプロイ対象がある場合、少なくとも dev 環境への自動デプロイパイプラインが動作している
