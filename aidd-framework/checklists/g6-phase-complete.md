# G6: Phase 完了マイルストーン

| 項目 | 内容 |
|------|------|
| 旧ゲート | G5 + G6 |
| 判断者 | PO + TL |
| タイミング | Phase の全 Epic 完了時 |
| 問い | **Phase のビジネス成果基準を満たしているか?** |

> **マイルストーンの役割:** 品質は各スキルが実行時に保証済み。このマイルストーンは、全 Epic の完了と Phase 成功基準の達成を統合確認する。

---

## Section 1: Epic 完了確認（旧 G5）

Epic ごとに以下を確認する。

### AC カバレッジ・トレーサビリティ

- [ ] Epic 仕様書の全 AC がチェック済み（チェックボックスが全て ON）
- [ ] AC → Task → PR のトレーサビリティが追跡可能である（各 AC-ID に対応する Task・PR 番号が特定できる）
- [ ] 全 AC-ID（`AC-ENNN-NN`）にテストが存在し、パスしている（AC-ID カバレッジ 100%）

### Task・PR 完了

- [ ] 全 Task が Done ステータス
- [ ] 全 PR がマージ済み

### E2E 検証・デモ

- [ ] Epic AC を直接検証する E2E テスト（E2E 検証 Task で定義）が通過している
- [ ] Epic 仕様書の「デモシナリオ」を PO が実行し、期待通りに動作することを確認した

### 非機能要件・BC 整合

- [ ] Epic 仕様書の非機能要件（定義されている場合）が目標値を達成している
- [ ] Epic が所属 BC 内に閉じている（BC 境界を越えた実装がないことを確認）

### feedback 管理

- [ ] 実装中に Epic 仕様書・設計の変更が発生した場合、feedback Issue に記録されている
- [ ] 実装中に規約ドキュメントへの追記が発生した場合、feedback Issue に記録されている
- [ ] この Epic に関連する feedback Issue が全てクローズされている
- [ ] feedback Issue の棚卸し: 各 Issue について対応済み / 次 Phase 送り / 見送りを判断した

---

## Section 2: Phase 完了確認（旧 G6）

全 Epic の完了確認後に実施する。

### 成功基準達成

- [ ] Phase 定義書の成功基準が全て E2E で検証済み
- [ ] 全 Epic が Section 1 の確認を通過済み
- [ ] 非機能要件（性能、セキュリティ、可用性）が目標値を達成

### マスタドキュメント更新

- [ ] マスタドキュメント（技術スタック、アーキテクチャ概要、DB 設計）が最新の状態に更新されている
- [ ] コンテキストマップの振り返り: BC 境界は適切だったか。実装を通じて境界の見直しが必要な箇所はないか
- [ ] `docs/domain/` 成果物（BC キャンバス、コンテキストマップ、ドメインイベント一覧）が最新状態に更新されている

### レトロスペクティブ

- [ ] プロセスの振り返りを実施した
- [ ] 改善アクションを Issue として起票し、次 Phase の Milestone に紐付けた

---

## 通過後アクション（`/aidd-next` が自動実行）

- [ ] Epic 仕様書のステータスを「完了」に更新
- [ ] Epic Issue をクローズ
- [ ] Phase 定義書のステータスを「完了」に更新
- [ ] Milestone にマイルストーン通過コメントを記録（`✅ G6 通過 (日付)`）
- [ ] Milestone をクローズ
