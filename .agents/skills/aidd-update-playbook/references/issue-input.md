# Step 1: 入力収集（Issue 起点モード）— 詳細手順

`--issue <番号>` 引数で指定された Issue を起点に、プレイブックの該当セクションを特定する。

## 入力

- 実行モード: Issue 起点（`--issue <番号>`）
- SKILL.md Step 0 で前提条件チェック済み

## 手順

### 1. FW 全体像・プレイブック全体像の読み込み

`step1-diff-analysis.md` の手順 1〜2 と同様に、FW 全体像とプレイブック全体像を読み込む。

> **CRITICAL:** Issue の内容だけでは該当セクションを正確に判定できない。FW とプレイブックの全体像を把握した上で判定すること。

### 2. Issue 内容の取得

```bash
gh issue view <番号> --json title,body,labels
```

**Issue が存在しない場合:**

- 「Issue #xxx が見つかりません」とエラーメッセージを表示して終了

### 3. Issue 内容の構造化

Issue の内容を以下の形式で整理する:

```markdown
## Issue サマリー

- **タイトル:** [Issue タイトル]
- **ラベル:** [ラベル一覧]
- **内容の要約:** [本文から1〜2文で要約]
- **関連するプレイブックの領域:**
  - セットアップに関するつまずき → 第2章
  - 開発フローに関する質問 → 第3章
  - エラー・トラブル報告 → 第4章
  - スキルの使い方に関する質問 → 第3章 or 第5章
  - FW の概念・原則に関する質問 → 第1章
```

### 4. 該当セクション判定

Issue の内容を分析し、プレイブック 5 章のいずれに該当するかを判定する。

**判定の指針:**

| Issue の特徴 | 該当する章 | 理由 |
|-------------|-----------|------|
| 「インストールで詰まった」「セットアップがうまくいかない」 | 02-setup | セットアップ手順の補足が必要 |
| 「スキルの使い方が分からない」「フローが不明」 | 03-development | 開発フローの対話例・説明の補足が必要 |
| 「エラーが出た」「動かない」「困った」 | 04-recovery-faq | FAQ・トラブルシューティングの追加が必要 |
| 「スキル一覧が古い」「ゲートの条件が変わった」 | 05-reference | リファレンスの更新が必要 |
| 「AIDD の考え方が分からない」「なぜこうなっている」 | 01-introduction | 概念説明の補足が必要 |

**複数の章に該当する場合:**

- 全ての該当章を一覧で提示する

### 5. 該当セクションなし判定

Issue の内容がプレイブックの5章のいずれにも該当しない場合:

- プレイブックのスコープ外の話題（コード実装の具体的な質問、特定プロジェクト固有の問題等）
- FW の内部設計に関する Issue（プレイブックは利用者向けであり、FW 内部設計は対象外）

**該当セクションなし判定時の出力:**

```markdown
## 該当セクション判定: 該当なし

この Issue はプレイブックのどのセクションにも該当しません。

**理由:** [具体的な理由]

プレイブックは FW の導入・利用者向けのガイドであり、
[Issue の内容の種別] はプレイブックのスコープ外です。
```

## 成果物

Issue サマリーと該当セクション判定結果。Step 2（影響セクション判定）に合流する。

## 完了時

判定結果をユーザーに提示し、確認を得る。確認後 Step 2 に進む（Step 2 は FW 差分起点と共通）。
