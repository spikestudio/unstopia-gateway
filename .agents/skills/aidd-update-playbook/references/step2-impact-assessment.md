# Step 2: 影響セクション判定 — 詳細手順

FW 全体像 + プレイブック全体像を把握した上で、FW の変更がプレイブックのどの章に影響するかを俯瞰的に判定する。

## 入力

- Step 1 で構造化された FW 変更サマリー
- Step 1 で読み込み済みの FW 全体像・プレイブック全体像

## 手順

### 1. 直接的影響の判定

FW 変更のファイルパスとプレイブック章の対応マップに基づいて、直接的に影響を受ける章を判定する:

| FW 変更パス | 直接影響する章 | 判定理由 |
|------------|-------------|---------|
| `aidd-framework/FRAMEWORK.md`（原則・ビジョン） | 01-introduction | プレイブックの前提が変わる |
| `aidd-framework/guides/`（セットアップ関連） | 02-setup | セットアップ手順が変わる |
| `skills/aidd-new-phase/`, `skills/aidd-new-epic/`, `skills/aidd-impl/` 等パイプラインスキル | 03-development | 開発フローの対話例が変わる |
| `skills/` 全般（エラーケース観点） | 04-recovery-faq | FAQ の追加・修正が必要になりうる |
| `skills/`（新規追加・削除）, `aidd-framework/process/milestones.md` | 05-reference | スキル早見表・ゲート早見表が変わる |

### 2. 間接的影響の判定

直接的影響に加えて、以下の観点で間接的影響を確認する:

| 間接的影響の観点 | 確認方法 |
|----------------|---------|
| **暗黙の前提の変化** | 「プレイブックの各章が暗黙に前提としていること」と「FW の変更」が矛盾しないか。例: 新しいロールの追加 → 第1章の「AI の役割」の範囲が変わる |
| **章間の整合性** | ある章の更新が他の章の記述と矛盾を生まないか。例: 第3章でスキル名が変わった → 第5章の早見表も変わる必要がある |
| **用語・概念の変化** | FW で用語が追加・変更された場合、プレイブック全体で古い用語が残っていないか |

### 3. 影響なし判定

以下のいずれかに該当する場合、「影響なし」と判定する:

- diff が FW 本体に影響しないファイルのみ（テストコード、CI 設定、`package.json` 等）
- diff が typo 修正・コメント追加等、プレイブックの記述内容に影響しない変更のみ
- diff がプレイブックの5章のいずれの対応領域にも該当しない

**影響なし判定時の出力:**

```markdown
## 影響セクション判定: 影響なし

FW の変更はプレイブックに影響しません。

**判定理由:**
- 変更は [変更内容の概要] のみであり、プレイブックの記述内容に影響しません
- 具体的には: [該当しない理由を章ごとに簡潔に]
```

### 4. 影響セクション一覧の提示

SKILL.md Step 2 に定義されたブリーフィング形式で提示する:

```markdown
## 影響セクション判定

### 影響あり

- **第3章（03-development.md）**: `/aidd-adr` スキルが新設されたため、開発フローの対話例に追加が必要。
  間接的影響: 横断スキルの利用タイミングの説明に `/aidd-adr` を含める必要がある
- **第5章（05-reference.md）**: スキル早見表に `/aidd-adr` の追加が必要

### 影響なし

- 第1章: AI の役割範囲に影響する変更なし
- 第2章: セットアップ手順に影響する変更なし
- 第4章: 既存の FAQ・トラブルシューティングに影響する変更なし

### 間接的影響の注記

- 第1章の「設計判断は人間が行う」の記述について: `/aidd-adr` は設計判断を **支援** するスキルであり、
  人間の判断を代替するものではないため、第1章の記述変更は不要と判定

この判定でよいですか？追加・除外したいセクションがあればお知らせください。
```

## 成果物

影響セクション一覧（根拠付き）。Step 3（セクション再執筆）の入力となる。

## 完了時

`references/briefing-spec.md` の Step 2 完了時の要素に従ってブリーフィングを実行する。ユーザーの承認を得てから Step 3 に進む。
