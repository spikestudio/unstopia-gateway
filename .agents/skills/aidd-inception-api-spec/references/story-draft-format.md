# ストーリードラフトフォーマット（API 版）

API 仕様から逆算したストーリードラフトの生成ガイド。`/aidd-new-epic` Step 0-2 の入力として使用できる形式で出力する。

## ストーリー生成ルール

> **ストーリー導出ルール**: 詳細は `skills/aidd-inception-mock/references/story-derivation-rules.md` を参照。
> 粒度・ペルソナ推定・AI補完の判定基準・情報源付記は全 Inception 子スキルで共通。

1. **1ストーリー = 1 API 操作（または1ユーザーフロー）** — 技術的な操作ではなくユーザーの目的で記述する
2. **ペルソナを明示** — API の利用者（フロントエンド開発者ではなくエンドユーザー視点）で記述する
3. **API 操作を隠蔽** — 「GET /users を呼び出す」ではなく「ユーザー一覧を確認する」と記述する
4. **根拠を付記** — どのエンドポイントから導出したかを明記する

## 出力テンプレート

```markdown
## 🔌 inception-api-spec セッション成果物

対象機能意図: [機能意図]
技術スタック: [REST/GraphQL/gRPC 等]
エンドポイント数: N
実施日: [yyyy-mm-dd]

---

## ストーリードラフト

> このセクションは /aidd-new-epic Step 0-2 の入力として使用します。

### 機能意図から導出したストーリー

- S1: As a **[ペルソナ]**, I want to [行動], so that [理由].
  根拠: [エンドポイント名]（[メソッド] [パス]）

- S2: As a **[ペルソナ]**, I want to [行動], so that [理由].
  根拠: [エンドポイント名]（[メソッド] [パス]）

### AI が補完提案したストーリー

- S3: As a **[ペルソナ]**, I want to [行動], so that [理由].
  根拠: [補完理由: 認証フロー / エラーハンドリング / 一括操作 等]

---

## API エンドポイント一覧（/aidd-new-epic 参照用）

| # | メソッド | パス | 概要 | 対応ストーリー |
|---|---------|------|------|--------------|
| 1 | GET | /[リソース] | [一覧取得] | S1 |
| 2 | POST | /[リソース] | [新規作成] | S2 |
| 3 | DELETE | /[リソース]/:id | [削除] | S3 |

---

## 未決定事項

API 設計過程で浮かび上がった未決定事項。Epic の AC 定義前に解消が必要。

| # | 未決定事項 | 影響するエンドポイント |
|---|-----------|---------------------|
| U1 | [未決定内容] | [エンドポイント名] |
| U2 | ... | ... |

---

## /aidd-new-epic への引き渡し方

/aidd-new-epic の Step 0-2 開始時に以下を提示する:

\`\`\`
inception-api-spec セッションの成果物があります。以下を Step 0-2 の入力として使用します:

[上記の成果物をここに貼付]

この内容をもとにストーリーを確認・補完し、AC を定義します。
\`\`\`
```

## /aidd-new-epic との対応関係

| inception-api-spec 成果物 | /aidd-new-epic での使用場所 |
|--------------------------|--------------------------|
| ストーリードラフト（機能意図から） | Step 0-2: ストーリー詳細化の入力（AI 補完を省略） |
| ストーリードラフト（AI 補完分） | Step 0-2: 追加ストーリー候補として確認 |
| API エンドポイント一覧 | Step 1: AC 導出時の API 参照 |
| リクエスト/レスポンス例 | Step 1: AC の入出力定義の参考 |
| 未決定事項 | Epic 仕様書「未決定事項」セクションに転記 |
