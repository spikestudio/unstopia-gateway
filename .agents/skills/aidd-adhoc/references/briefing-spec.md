# ブリーフィング仕様: /aidd-adhoc

単発作業の実装完了後、5 要素ブリーフィングを実行する。種別に応じて重点が異なる。

## 5 要素ブリーフィング

| # | 要素 | 内容 |
|---|------|------|
| 1 | 成果物の全体像 | 種別、変更ファイル、変更量 |
| 2 | AC との対応 | 修正がバグの根本原因を解決しているか（fix/hotfix）、Task の AC を満たしているか（task） |
| 3 | AI の判断と根拠 | 修正方針の選択理由、影響範囲の判断 |
| 4 | AI 補完の明示 | Task 定義にない判断を AI が補完した箇所 |
| 5 | 不確実性の表明 | 修正の副作用リスク、テストカバレッジの不足箇所 |

## 種別ごとの重点

| 種別 | 重点 |
|------|------|
| fix | 根本原因の特定根拠、リグレッションテストのカバー範囲、影響範囲の評価 |
| hotfix | 修正の最小性、緊急度に見合った対応か、後続で必要な追加対応 |
| task | 親 Epic との整合性、既存 Task とのスコープ重複がないか |
| chore | 変更の最小性、副作用がないことの確認 |
