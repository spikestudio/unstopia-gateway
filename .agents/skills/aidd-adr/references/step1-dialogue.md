# Step 1: 対話による ADR 内容引き出し — 詳細手順

`agents/aidd-architect.md` の ADR 問いリストに従ってユーザーと対話し、ADR の内容を引き出す。

## 入力

- Step 0 で決定した ADR タイトル
- `agents/aidd-architect.md`（ADR 問いリスト）

## 手順

### 1. aidd-architect サブエージェントの参照

`agents/aidd-architect.md` §ADR 作成時の問いリスト を読み込み、以下の 5 問を順にユーザーに問いかける。

> **独自の問いリストを持たない。** `agents/aidd-architect.md` の問いリストを参照し、そこに記載された質問をそのまま使用する。問いリストが変更された場合、このステップは自動的に最新の問いに従う。

### 2. 5 問の対話

`agents/aidd-architect.md` の問いリスト（背景・選択肢・トレードオフ・決定内容・影響範囲）を 1 問ずつ順に問いかける。

各問いに対して行うこと:

- ユーザーの回答を引き出す
- 回答が不明確・不十分な場合は掘り下げ質問をする
- 選択肢は最低 2 つ提示されること（1 つしかない場合は「現状維持」を追加する）
- AI 自信度が低い箇所は `<!-- AI-UNCERTAIN: [理由] -->` を付記する

### 3. ADR 下書きの生成

5 問への回答が揃ったら、`aidd-framework/templates/adr.md` の構造に沿って ADR 下書きを生成し、ユーザーに提示する。

```markdown
## 下書き確認: [ADR タイトル]

[下書き内容]

この内容で Step 2（採番・保存）に進んでよいですか？
修正がある場合は指摘してください。
```

## 成果物

ADR の下書き（背景・選択肢・トレードオフ・決定内容・影響範囲が揃った状態）
