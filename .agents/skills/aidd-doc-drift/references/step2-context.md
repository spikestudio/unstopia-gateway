# Step 2: コンテキスト収集 — 詳細手順

スキル同梱のスクリプトでドキュメント一覧を取得し、エントリポイントドキュメントと変更ファイルの中身を読んで推論に必要なコンテキストを揃える。

## 2-1. scan-docs.sh の実行

```bash
bash "${CLAUDE_PLUGIN_ROOT}/scripts/scan-docs.sh"
```

出力（ファイルパス + 冒頭 15 行）を読んで、プロジェクトにどんなドキュメントが存在するかの**鳥瞰図**を把握する。

**`docs/` が存在しない場合 / `.md` ファイルが0件の場合:**

```
ドキュメントファイルが見つかりませんでした。
判断に使えるコンテキストが限られるため、推論精度が低い可能性があります。
引き続き変更ファイルの内容とプロジェクト構造から推論を試みます。
```

上記メッセージをユーザーに伝えた上で 2-2 に進む（スキルを停止しない）。

## 2-2. エントリポイントドキュメントの読み込み

以下を優先順で探し、存在するものを読む（複数あれば全て読む）:

| 優先順 | ファイル | 役割 |
|-------|---------|------|
| 1 | `CLAUDE.md` | プロジェクト規約・スキル構成・禁止事項 |
| 2 | `README.md` / `README` | プロジェクト概要・構成 |
| 3 | `.cursor/rules` | Cursor 向けプロジェクト規約 |
| 4 | `AGENTS.md` | エージェント向け指示 |

**どれも存在しない場合:**

```
エントリポイントドキュメント（CLAUDE.md, README.md 等）が見つかりませんでした。
プロジェクト固有のコンテキストなしで推論します。精度が低い可能性があります。
```

## 2-3. 変更ファイルの中身を読む

Step 1 で特定した走査対象（PR または コミット）の変更ファイルを取得し、実際に読む。

```bash
# PR モードの場合（PR 番号が判明している場合）
gh pr diff <PR番号>

# コミットモードの場合
git diff <範囲>
```

**読む対象の優先順位:**

1. `git diff --name-only` で変更ファイル一覧を取得
2. 変更量が多い場合（20ファイル超）は全読みせず、ファイル名と diff のサマリーから優先度を判断して重要なものを選択的に読む
3. 削除ファイル・リネームファイルは特に注意して記録する（参照が残存しやすいため）

## 2-4. 推論の実行（4段階フロー）

収集したコンテキストをもとに影響ドキュメントを推論する。

### Stage 1: コンテキスト整理

以下を整理する:

- scan-docs.sh の出力から「どんなドキュメントが存在するか」
- エントリポイント doc から「このプロジェクトの規約・構成」
- diff から「何がどう変わったか（追加・削除・変更・リネームの区別）」

### Stage 2: 変更の意味理解

変更ファイルの中身を読んで、変更の**意味**を把握する:

| 変更の種類 | 意味理解のポイント |
|----------|----------------|
| 新規ファイル追加 | 何の機能・概念が追加されたか |
| ファイル削除 | 何が廃止・統合されたか |
| リネーム | 何が改名されたか（旧名称がドキュメントに残存しやすい）|
| 内容変更 | 何の仕様・動作が変わったか |

### Stage 3: 影響ドキュメントの推論

変更の意味と scan-docs 出力を照合し、影響を受ける可能性があるドキュメントを列挙する。各候補に**推論根拠**を付与する。

推論の観点:

- 変更対象の名称（ファイル名・クラス名・スキル名等）が参照されているドキュメントはないか
- 変更によって「目次・一覧・パイプライン図」等に記載漏れが生じていないか
- 廃止・リネームによって「存在しないものへの参照」が残存していないか
- 新規追加によって「本来存在すべきドキュメント」が作られていないか

### Stage 4: 候補の提示

推論結果をユーザーに提示して確認を得てから Step 3 に進む:

```
## 影響ドキュメント候補

以下のドキュメントに乖離がある可能性があります。Step 3 で詳細確認します。

| # | ドキュメント | 推論根拠 |
|---|------------|---------|
| 1 | aidd-framework/FRAMEWORK.md | 新規追加スキル `/aidd-xxx` が横断スキル表に未記載の可能性 |
| 2 | CLAUDE.md | スキルパイプライン記述が変更後の構成を反映していない可能性 |
| ...

この候補で Step 3（詳細確認）に進みますか？追加・削除があればお知らせください。
```
