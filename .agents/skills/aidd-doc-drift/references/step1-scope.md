# Step 1: スコープ決定 — 詳細手順

`$ARGUMENTS` を解析し、走査対象（PR または コミット）を決定してユーザーに確認する。

---

## 1. 引数解析

`$ARGUMENTS` の内容に応じて走査モードを決定する。

| 引数パターン | モード | 動作 |
|-------------|--------|------|
| 省略（空） | PR 履歴モード | 直近 10 件の PR を走査 |
| 数字のみ（例: `5`） | PR 履歴モード | 指定件数の PR を走査 |
| `--since YYYY-MM-DD` | PR 履歴モード | 指定日付以降の PR を走査 |
| `--commits HEAD~N` | コミット履歴モード | 指定範囲のコミット履歴を走査 |

引数が上記いずれにも該当しない場合は、エラーメッセージと使用例を提示してスキルを停止する。

```
エラー: 引数を認識できませんでした。
使用例:
  /aidd-doc-drift           # 直近 10 件の PR
  /aidd-doc-drift 5         # 直近 5 件の PR
  /aidd-doc-drift --since 2026-04-01
  /aidd-doc-drift --commits HEAD~10
```

---

## 2. PR 履歴モード

### 2-1. PR 一覧を取得する

**件数指定なし（デフォルト 10 件）:**

```bash
gh pr list --state merged --limit 10 --json number,title,mergedAt,files
```

**件数指定あり（例: 5）:**

```bash
gh pr list --state merged --limit 5 --json number,title,mergedAt,files
```

**日付指定（例: --since 2026-04-01）:**

```bash
gh pr list --state merged --limit 100 --json number,title,mergedAt,files \
  | jq '[.[] | select(.mergedAt >= "2026-04-01")]'
```

日付フィルタは `gh pr list` がネイティブに対応していないため、jq でフィルタする。

### 2-2. 未マージコミットのチェック（フォールバック）

PR 一覧の取得後、以下のコマンドで未マージコミットが存在するか確認する:

```bash
git log main..HEAD --oneline
```

- **出力が空** → 未マージコミットなし。PR 走査のみで進む。
- **出力あり** → 未マージコミットが存在する。以下のメッセージでユーザーに確認を求める:

```
未マージの作業が検出されました。コミット履歴も走査しますか？

検出されたコミット:
  [git log main..HEAD --oneline の出力]

[y] 走査する  [n] PR 履歴のみで進む
```

ユーザーが「y」を選択した場合、コミット履歴モードを追加実行し、PR 走査結果と合わせてスコープとする。

---

## 3. コミット履歴モード（`--commits HEAD~N` 指定時）

```bash
git log HEAD~N..HEAD --oneline
```

で対象コミット一覧を取得する。各コミットの変更ファイルは後続の Step 2 で参照する。

---

## 4. ユーザー確認

走査対象の一覧を以下の形式で提示し、「この範囲で進めますか？」と確認を得てから Step 2 に進む。

### PR 履歴モードの場合

```
以下の PR を走査対象とします:

  #123  feat: ES-028 ドキュメント連動（2026-04-07 マージ）
  #120  feat: ES-027 gitflow 整合（2026-04-05 マージ）
  ...

この範囲で進めますか？ [y/n]
```

### コミット履歴モードの場合

```
以下のコミットを走査対象とします:

  abc1234  feat: TASK-170 Step 1 スコープ決定ロジック
  def5678  fix: scan-docs.sh の改行処理を修正
  ...

この範囲で進めますか？ [y/n]
```

### PR + コミット履歴の複合スコープの場合

PR 一覧とコミット一覧をそれぞれ分けて提示する。

---

## 5. 確認結果による分岐

- **「y」または明確な承認** → 決定したスコープを保持して Step 2 に進む
- **「n」またはキャンセル** → 走査範囲の再指定を案内してスキルを停止する:

```
走査を中止しました。
別の範囲で実行する場合は引数を変えて再実行してください。
例: /aidd-doc-drift 5 / /aidd-doc-drift --since 2026-04-01
```
