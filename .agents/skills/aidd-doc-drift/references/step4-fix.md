# Step 4: ブリーフィング・修正実行 — 詳細手順

Step 3 で確認した乖離をユーザーにブリーフィングし、承認を得てから修正を実行する。

## 4-1. ブリーフィング

乖離を2種類に分類して提示する。

```
## ドキュメント乖離レポート

### 更新が必要なドキュメント（N 件）
既存ドキュメントの修正が必要なもの。

| # | ドキュメント | 箇所 | 問題 | 修正案 |
|---|------------|------|------|--------|
| 1 | aidd-framework/FRAMEWORK.md | L544 横断スキル表 | /aidd-doc-drift が未記載 | 1行追加 |
| 2 | CLAUDE.md | §スキルパイプライン | 旧スキル名 /aidd-xxx が残存 | /aidd-yyy に更新 |

### 新規作成が必要なドキュメント（M 件）
変更に対応するドキュメントが存在しないもの。

| # | 作成先 | 内容 | 理由 |
|---|--------|------|------|
| 1 | aidd-framework/guides/doc-drift-guide.md | /aidd-doc-drift の使い方ガイド | スキル新設に伴い利用者向けガイドが必要 |

### スキップ（乖離なし）
- docs/conventions/naming.md — 変更の影響なし

---

修正してよいですか？個別にスキップしたい項目があればお知らせください。
```

**ユーザーが一部をスキップした場合:** スキップされた項目を記録し、修正対象から除外する。

## 4-2. 修正実行

承認を得た項目を順に修正する。**自動修正禁止 — 必ず承認後に実行する。**

### 更新の場合

対象ドキュメントを読み込み、乖離箇所を修正する。修正前後の差分をユーザーに確認できるよう、変更内容を明示しながら進める。

### 新規作成の場合

新規ドキュメントの内容を生成する前に、ファイルパス・構成・主な内容をユーザーに提示して確認を得る。

```
以下の内容で作成してよいですか？

ファイル: aidd-framework/guides/doc-drift-guide.md
構成:
- 概要・使いどころ
- 基本的な使い方（引数パターン）
- 注意事項

よろしければ作成します。
```

## 4-3. 修正後の確認

全修正が完了したら、変更ファイルの一覧を提示する。

```
## 修正完了

### 変更したファイル
- aidd-framework/FRAMEWORK.md（更新）
- CLAUDE.md（更新）
- aidd-framework/guides/doc-drift-guide.md（新規作成）

### スキップした項目
- なし

Step 5（セルフレビュー・コミット）に進みます。
```
