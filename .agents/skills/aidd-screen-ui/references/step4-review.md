# Step 4: G3 レビューゲート・反復フロー — 詳細手順

## 4-1. G3 Section 2 チェックリストの提示

以下のチェックリストを提示し、PO/TL がレビューを実施する:

```
## G3 Section 2 チェックリスト: 画面設計承認

Epic: ES-NNN
対象画面数: N 件

### チェック項目

- [ ] 全画面の UIKit コンポーネントが選定されている
  （または「カスタム実装」として記録されている）
- [ ] レスポンシブレイアウトが定義されている
  （または「対応不要」として明記されている）
- [ ] アノテーション（コンポーネント名・Props・インポートパス）が付与されている

### 成果物確認先

- 設計ドキュメント: docs/design/screen-ui-[slug]/
- 対象 SCR-ID: [一覧]

全チェック項目が OK の場合: 「PASS」と入力してください。
レビュー指摘がある場合: 指摘内容を入力してください。
```

## 4-2. レビュー指摘の記録

ユーザーからレビュー指摘が入力された場合、以下の形式で記録する:

```
## レビュー指摘の記録

指摘 #1:
- 対象: [SCR-ID] [画面名] の [要素名]
- 内容: [指摘内容]
- ステータス: 未対応

「対応」「対応不要」「保留」のいずれかでステータスを設定してください。
対応内容を入力してから次の指摘に進みます。
```

## 4-3. 指摘への対応

各指摘に対して以下のステータスを設定できる:

| ステータス | 意味 | 次のアクション |
|----------|------|-------------|
| **対応済み** | 指摘内容を反映した | 設計ドキュメントを更新して次の指摘へ |
| **対応不要** | 指摘は不要と判断した（理由を記録） | 理由を記録して次の指摘へ |
| **保留** | 後で対応する（理由と期限を記録） | 理由・期限を記録して次の指摘へ |

### 「対応済み」の場合の設計ドキュメント更新

```
指摘 #N を「対応済み」としました。
対応内容: [対応内容の説明]

以下のドキュメントを更新します:
- docs/design/screen-ui-[slug]/[SCR-ID]-[screen-slug].md

更新内容を確認してよいですか？（OK）
```

## 4-4. G3 PASS 判定

全指摘が「対応済み」または「対応不要」になったら G3 PASS として承認フローに進む:

```
## G3 Section 2: PASS ✅

全レビュー指摘の対応が完了しました。

### 指摘サマリー
| # | 指摘内容 | ステータス |
|---|---------|----------|
| 1 | [内容] | ✅ 対応済み |
| 2 | [内容] | — 対応不要 |

### 生成成果物
- docs/design/screen-ui-[slug]/（N 件）

次のステップ:
→ `/aidd-decompose-epic` で Task 分解（G4）に進む
→ `/aidd-next` で G3 マイルストーン判定
```

## 4-5. G3 FAIL / 保留がある場合

保留の指摘が残っている場合は G3 PASS と見なさない:

```
## G3 Section 2: 保留あり ⚠️

保留の指摘が N 件あります。
保留の指摘を「対応済み」または「対応不要」にしてから G3 PASS を宣言してください。

保留中の指摘:
- 指摘 #N: [内容]（理由: [理由]、期限: [期限]）
```
