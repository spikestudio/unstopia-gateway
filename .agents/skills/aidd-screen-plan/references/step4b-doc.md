# Step 4b: 優先度・MVPスコープ・Epic紐付け + ドキュメント生成 — 詳細手順

AC-E052-10（優先度の対話設定）、AC-E052-11（MVPスコープの設定・明記）、AC-E052-12（担当 Epic の紐付け）、AC-E052-13（未割当画面の記録）、AC-E052-14（統一フォーマットでの画面一覧ドキュメント生成）を満たす詳細手順を定義する。

## 入力

- Step 4a（step4a-id.md）で確定した全画面リスト（SCR-ID・画面名・種別・導出元 AC 参照・AC カバレッジサマリー）
- Epic 仕様書（対象 Epic の識別子 `ES-NNN` と Epic 名を取得するために参照）

## 4b-1. 優先度の対話設定（AC-E052-10）

各画面の優先度（MUST / SHOULD / COULD）をユーザーと対話的に設定する。デフォルト値は `MUST`。

### 優先度の定義

| 優先度 | 意味 |
|--------|------|
| MUST | MVP に必須。この画面がなければリリースできない |
| SHOULD | 重要だが MVP には含めなくても良い |
| COULD | あれば良い。優先度が低い |

### 優先度設定の提示フォーマット

全画面の優先度一覧をデフォルト値（全て MUST）で提示し、変更を受け付ける:

```markdown
## Step 4b-1: 優先度の設定

各画面のデフォルト優先度は MUST です。変更したい場合は「N番は SHOULD」のように指示してください。

| # | SCR-ID | 画面名 | 優先度 |
|---|--------|--------|--------|
| 1 | SCR-001 | [画面名] | MUST |
| 2 | SCR-002 | [画面名] | MUST |
| 3 | SCR-003 | [画面名] | MUST |

変更があれば指示してください。なければ「OK」または「進めて」で確定します。
```

### 対話操作の受け付け

```
例: 「3番は SHOULD」
例: 「エラー画面は COULD にしてください」
例: 「5、6番は COULD」
```

処理:

1. 指定された画面の優先度を更新する
2. 更新後のリストを再提示する
3. 追加変更がなければユーザーの明示的な承認（「OK」「進めて」等）を得て確定する

### バリデーション

| ケース | 対応 |
|--------|------|
| MUST / SHOULD / COULD 以外が指定された | 「指定された優先度は無効です。MUST / SHOULD / COULD のいずれかで指定してください。デフォルト値（MUST）を適用します。」と通知し、再入力を促す |
| 存在しない番号が指定された | 「N番はリストにありません。」と案内し、再入力を促す |

## 4b-2. MVPスコープの設定（AC-E052-11）

各画面の MVPスコープ（in / out）を設定する。

### MVPスコープのデフォルト値

優先度に基づいて以下のデフォルト値を適用する:

| 優先度 | デフォルト MVPスコープ |
|--------|----------------------|
| MUST | in |
| SHOULD | in |
| COULD | out |

### MVPスコープ設定の提示フォーマット

優先度確定後に MVPスコープをデフォルト値で提示し、変更を受け付ける:

```markdown
## Step 4b-2: MVPスコープの設定

優先度をもとにデフォルト値を設定しました。変更したい場合は「N番は out」のように指示してください。

| # | SCR-ID | 画面名 | 優先度 | MVPスコープ |
|---|--------|--------|--------|------------|
| 1 | SCR-001 | [画面名] | MUST | in |
| 2 | SCR-002 | [画面名] | SHOULD | in |
| 3 | SCR-003 | [画面名] | COULD | out |

変更があれば指示してください。なければ「OK」または「進めて」で確定します。
```

### MVPスコープ変更の対話操作

```
例: 「2番は out」
例: 「すべての SHOULD を out にしてください」
```

処理:

1. 指定された画面の MVPスコープを更新する
2. 更新後のリストを再提示する
3. ユーザーの明示的な承認を得て確定する

### MVPスコープ内画面の抽出（AC-E052-11）

確定後に MVPスコープ `in` の画面を抽出し、サマリーとして表示する:

```markdown
### MVPスコープサマリー

- MVPスコープ in: N 件
- MVPスコープ out: M 件

**MVP対象画面一覧（in のみ）:**

| SCR-ID | 画面名 | 優先度 |
|--------|--------|--------|
| SCR-001 | [画面名] | MUST |
| SCR-002 | [画面名] | SHOULD |
```

このサマリーは生成するドキュメントの「MVPスコープサマリー」セクションに含める。

## 4b-3. 担当 Epic の紐付け（AC-E052-12・AC-E052-13）

各画面に担当 Epic の識別子（`ES-NNN`）を紐付ける。担当 Epic が未定の場合は「未割当」として記録する。

### 担当 Epic 設定の提示フォーマット

デフォルト値として、このスキルを実行した対象 Epic（引数で指定された Epic）を全画面に適用し、変更を受け付ける:

```markdown
## Step 4b-3: 担当 Epic の紐付け

デフォルトとして対象 Epic（ES-NNN）を全画面に設定しました。
他の Epic を担当する画面や担当が未定の画面は変更してください。

| # | SCR-ID | 画面名 | 担当 Epic |
|---|--------|--------|----------|
| 1 | SCR-001 | [画面名] | ES-NNN |
| 2 | SCR-002 | [画面名] | ES-NNN |
| 3 | SCR-003 | [画面名] | ES-NNN |

変更があれば指示してください（例: 「3番は ES-055」「4番は未割当」）。
なければ「OK」または「進めて」で確定します。
```

### 担当 Epic 変更の対話操作

```
例: 「3番は ES-055」
例: 「4番は未割当」
例: 「ユーザー編集画面は ES-056 にしてください」
```

処理:

1. 指定された画面の担当 Epic を更新する（`ES-NNN` 形式 または `未割当`）
2. 更新後のリストを再提示する
3. ユーザーの明示的な承認を得て確定する

### 「未割当」の記録（AC-E052-13）

担当 Epic が未定の画面は「未割当」として記録する。後から更新できるよう、生成するドキュメントの該当セルに「未割当」と明記する。

### 担当 Epic のバリデーション

| ケース | 対応 |
|--------|------|
| `ES-NNN` 形式でない文字列が指定された（「未割当」を除く） | 「指定された Epic 識別子の形式が不正です。`ES-NNN` 形式（例: ES-010）または `未割当` で指定してください。」と案内し、再入力を促す |

## 4b-4. ドキュメント生成（AC-E052-14）

Step 4a・4b-1〜4b-3 の全確定データを使用して、`docs/design/screen-list-[epic-slug].md` を生成する。

### epic-slug の決定

ファイル名の `[epic-slug]` は Epic 仕様書のファイル名（`docs/requirements/ES-NNN-[slug].md` の `[slug]` 部分）から取得する。

例:

- `docs/requirements/ES-052-screen-plan.md` → `screen-list-screen-plan.md`
- `docs/requirements/ES-010-user-management.md` → `screen-list-user-management.md`

### 生成フォーマット（AC-E052-14）

`references/screen-list-template.md` に従って以下の内容を含むドキュメントを生成する:

```markdown
<!-- 配置先: docs/design/screen-list-[epic-slug].md にコピーして使用する -->
<!-- 生成スキル: /aidd-screen-plan -->
<!-- 後続スキル: /aidd-screen-spec（SCR-ID・元AC参照 を入力として使用） -->

# 画面一覧: [Epic名]（[ES-NNN]）

| 項目 | 内容 |
|------|------|
| 対象 Epic | ES-NNN |
| 生成日 | yyyy-mm-dd |
| 生成スキル | /aidd-screen-plan |

## 画面一覧

| SCR-ID | 画面名 | 種別 | 優先度 | MVPスコープ | 担当 Epic | 元 AC 参照 |
|--------|--------|------|--------|------------|----------|-----------|
| SCR-001 | [画面名] | [種別] | [MUST/SHOULD/COULD] | [in/out] | [ES-NNN/未割当] | [AC-ID / 手動追加 / ドメインモデル] |
| ...    | ...    | ...  | ...    | ...        | ...      | ...       |

## MVPスコープサマリー

- MVPスコープ in: N 件
- MVPスコープ out: M 件

### MVP対象画面一覧（in のみ）

| SCR-ID | 画面名 | 優先度 |
|--------|--------|--------|
| SCR-001 | [画面名] | MUST |
| ...    | ...    | ...   |

## AC カバレッジサマリー

| 項目 | 件数 |
|------|------|
| 対象 Epic の全 AC 数 | N 件 |
| 画面として表現された AC 数 | M 件 |
| 未表現の AC 数 | K 件 |

### 未表現の AC（参考）

| AC-ID | 内容 | 画面化しない理由 |
|-------|------|----------------|
| AC-ENNN-NN | [AC内容の概要] | [バックエンド処理のため / 別画面に統合済み / 等] |

## 除外した画面候補（参考）

| 画面名（案） | 除外理由 |
|------------|---------|
| [除外した画面名] | [除外理由] |
```

### 生成手順

1. `docs/design/` ディレクトリが存在しない場合は作成する
2. `screen-list-[epic-slug].md` ファイルを生成する
3. 全確定データ（SCR-ID・画面名・種別・優先度・MVPスコープ・担当 Epic・元 AC 参照・AC カバレッジサマリー）を埋め込む
4. 「生成日」には今日の日付（`yyyy-mm-dd` 形式）を記入する
5. 「除外した画面候補」セクションには Step 1・Step 2 でユーザーが除外した画面を記録する（除外された画面がない場合はテーブルを空にする）

### ドキュメント生成後の確認提示

生成したドキュメントの内容をユーザーに提示して確認を求める:

```markdown
## ドキュメント生成完了

`docs/design/screen-list-[epic-slug].md` を生成しました。

### 生成サマリー
- 画面数: N 件（MUST: N / SHOULD: N / COULD: N）
- MVPスコープ in: N 件 / out: N 件
- 担当 Epic 未割当: N 件（あれば記載、0 件なら省略）
- AC カバレッジ: M/N AC が画面として表現済み

生成されたドキュメントの内容を確認してください。
問題なければ「OK」または「進めて」でブリーフィングに進みます。
```

## エラーケース

| ケース | 条件 | 対応 |
|--------|------|------|
| `docs/design/` への書き込みに失敗した | ディレクトリ作成・ファイル書き込みでエラーが発生した | エラー内容を表示し、権限やパスを確認するよう案内する |
| ファイルが既に存在する | 同名の `screen-list-*.md` が既に存在する | 「既存のファイルが存在します: [パス]。上書きしますか？」と確認し、ユーザーの明示的な承認を得てから上書きする |

## 完了時

以下の情報を Step 5（ブリーフィング）に渡す:

- 生成したファイルのパス（`docs/design/screen-list-[epic-slug].md`）
- 画面数の内訳（MUST / SHOULD / COULD 別）
- MVPスコープの内訳（in / out 別）
- 担当 Epic 未割当の画面数（あれば）
- AC カバレッジサマリー（全 AC 数・画面化済み数・未表現数）
