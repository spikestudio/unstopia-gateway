# Step 4a: 画面 ID 付与・AC トレーサビリティ — 詳細手順

AC-E052-07（SCR-NNN 形式の一意 ID 付与）、AC-E052-08（導出元 AC-ID の記録）、AC-E052-09（AC カバレッジサマリーの生成）を満たす詳細手順を定義する。

## 入力

- Step 3 で確定した全画面候補リスト（画面名・導出元 AC・確定種別）
- Step 0 で取得した既存画面 ID の最大値（重複回避に使用）
- Epic 仕様書の全 AC 一覧（カバレッジ計算に使用）

## 4a-1. 画面 ID の採番（AC-E052-07）

確定した画面候補リストの各画面に `SCR-NNN` 形式の一意 ID を連番で付与する。

### 採番ルール

1. Step 0 で確認した既存画面 ID の最大値を取得する
   - 既存ファイルが存在しない場合: `SCR-001` から開始する
   - 既存ファイルが存在する場合: 既存の最大 ID の次の番号から開始する
2. 確定した画面候補リストを提示された順（# 順）に並べる
3. 先頭から連番で `SCR-NNN`（3 桁ゼロ埋め）を付与する
4. 同一ドキュメント内での重複がないことを検証する（重複する場合は次の連番を自動付与）

### 採番例

| 既存最大 ID | 新規採番開始 |
|-----------|-----------|
| なし（初回） | SCR-001 |
| SCR-005 | SCR-006 |
| SCR-012 | SCR-013 |

### 採番結果の内部記録

各画面について以下を記録する:

```
{
  SCR-ID: "SCR-NNN",
  画面名: "[画面名]",
  確定種別: "[種別]",
  導出元AC: ["AC-ENNN-NN", ...]  // 複数の場合は配列で保持
}
```

## 4a-2. 導出元 AC-ID の記録（AC-E052-08）

各画面の導出元 AC-ID を記録する。1 つの AC から複数画面が導出された場合も、全ての画面に同じ AC-ID を記録する。

### 記録ルール

| ケース | 記録形式 | 例 |
|--------|---------|-----|
| 1 つの AC から導出 | AC-ID を 1 件記録 | `AC-E052-01` |
| 複数の AC から導出（統合された場合） | カンマ区切りで全 AC-ID を記録 | `AC-E052-01, AC-E052-02` |
| 手動追加（AC 以外から追加） | `手動追加` として記録 | `手動追加` |
| ドメインモデルから補完 | `ドメインモデル` として記録 | `ドメインモデル` |

### 1:N の扱い（1 つの AC から複数画面が導出された場合）

同一の AC-ID を、導出された全ての画面の「元 AC 参照」カラムに記録する。

例: `AC-E052-03` から「ユーザー一覧画面」と「ユーザー検索画面」が導出された場合:

| SCR-ID | 画面名 | 元 AC 参照 |
|--------|--------|-----------|
| SCR-001 | ユーザー一覧画面 | AC-E052-03 |
| SCR-002 | ユーザー検索画面 | AC-E052-03 |

## 4a-3. AC カバレッジサマリーの生成（AC-E052-09）

Epic 仕様書の全 AC と、画面一覧の「元 AC 参照」カラムを照合し、カバレッジを集計する。

### 集計ルール

1. Epic 仕様書から全 AC-ID を抽出する（`AC-ENNN-NN` 形式）
1. 画面一覧の「元 AC 参照」カラムに記録された AC-ID を収集する
1. 以下の 3 つの数値を算出する:

| 項目 | 算出方法 |
|------|---------|
| 全 AC 数 | Epic 仕様書の AC-ID の総数 |
| 画面として表現された AC 数 | 画面一覧の「元 AC 参照」に 1 件以上の AC-ID が記録されている AC の数（重複を除く） |
| 未表現の AC 数 | 全 AC 数 − 画面として表現された AC 数 |

1. 「未表現の AC」については、どの AC が画面化されていないかを一覧表示する

### 未表現 AC の判定

以下に該当する AC-ID を「未表現」とする:

- 画面一覧の「元 AC 参照」カラムに一度も出現しない AC-ID
- `手動追加` や `ドメインモデル` は AC 由来ではないため集計対象外とする

### カバレッジサマリーのフォーマット

```markdown
## AC カバレッジサマリー

| 項目 | 件数 |
|------|------|
| 対象 Epic の全 AC 数 | N 件 |
| 画面として表現された AC 数 | M 件 |
| 未表現の AC 数 | K 件 |

### 未表現の AC（参考）

| AC-ID | 内容 | 画面化しない理由 |
|-------|------|----------------|
| AC-ENNN-NN | [AC 内容の概要] | [バックエンド処理のため / 別画面に統合済み / 等] |
```

未表現の AC がある場合は、その AC の内容と「画面化しない理由」をユーザーと確認する。

## 4a-4. 完了時の出力

Step 4b（優先度・MVPスコープ・Epic紐付け）に以下を渡す:

- SCR-ID が付与された全画面リスト（画面名・種別・SCR-ID・元 AC 参照）
- AC カバレッジサマリー（全 AC 数・画面化済み数・未表現数・未表現 AC の一覧）

## エラーケース

| ケース | 条件 | 対応 |
|--------|------|------|
| SCR-ID が重複する | 同一ドキュメント内に同じ SCR-ID が存在する | 次の連番を自動付与して重複を解消し、ユーザーに通知する |
| AC-ID の形式が不正 | `AC-ENNN-NN` 形式に合致しない文字列が導出元 AC に含まれる | 「形式が不正な AC-ID が含まれています: [値]」と警告し、手動修正を促す |
| Epic 仕様書に AC が存在しない | AC 一覧が空 | カバレッジサマリーを「全 AC 数: 0 件、未表現: 0 件」として出力し、警告を表示する |
