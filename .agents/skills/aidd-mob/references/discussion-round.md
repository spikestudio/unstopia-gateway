# 議論ラウンド手順

mob セッションの議論ラウンドの詳細手順。

## ラウンドの定義

**1ラウンド** = 参加する全ペルソナが、前ラウンド（または初回発言）の内容を参照して、反論・補足・深掘りを行うサイクル。

- **最低ラウンド数**: 2（1ラウンドで収束させない。内容が豊かでも必ず2ラウンド実行する）
- **推奨上限**: 5ラウンド（ユーザーが明示的に続行を選択した場合は超過可）
- **初回発言はラウンドとして数えない**（リサーチ→初回発言 → ラウンド1 → ラウンド2 ... の順）

### 2ラウンド未満での収束禁止

ラウンド1終了時にユーザーが「B) 収束」を選択した場合、収束を拒否してラウンド2を強制実行する:

```
⚠️ 最低2ラウンド実行が必要です
ラウンド1で収束させることはできません。ラウンド2を実行してから収束を選択してください。

🔄 ラウンド 2 開始
```

ラウンド2以降は、ユーザーが「B) 収束」を選択した時点で Step 4（成果物出力）に進む。

## ラウンドの実行手順

### ラウンド開始時（AC-E070-01: 前ラウンドハイライト再提示）

各ラウンド開始時に、前ラウンドの合意点・対立点・未決定事項を **必ず再提示**してから
各ペルソナに発言させる。コンテキスト肥大化を防ぎつつ議論の継続性を保つための必須ルール。

```
🔄 ラウンド N 開始

📌 前ラウンド（ラウンド N-1）のハイライト:
  ✅ 合意点: [全ペルソナが同意した内容（箇条書き 1〜3件）]
  ⚡ 対立点: [意見が分かれている点（箇条書き 1〜2件）]
  ❓ 未決定事項: [まだ答えが出ていない問い（箇条書き 1〜2件）]

このハイライトを踏まえて各ペルソナが発言します...
```

> ラウンド1の場合は「前ラウンド」ではなく「初回発言」のハイライトを提示する。

### 各ペルソナの発言フォーマット

```
**[ペルソナ名]（[役割]）:**

[前ラウンドの発言への参照・反論・補足・深掘り]

[新たに提起する問いや視点]

根拠: [リサーチ情報源 or 前ラウンドのどの発言への応答か]
```

### ペルソナ発言の品質基準

各ペルソナの発言は以下を満たすこと:

| 基準 | 説明 |
|------|------|
| **前ラウンド参照** | 他ペルソナの発言を具体的に引用または言及する |
| **新規性** | 前ラウンドの自分の発言の繰り返しではなく、議論を進展させる内容を含む |
| **根拠付き** | 主張の根拠（リサーチ情報源・業界知見・論理）を示す |
| **具体性** | 抽象的な意見ではなく、具体的な制約・リスク・代替案を示す |

### ユーザー介入

ラウンド中・ラウンド間を問わず、ユーザーはいつでも発言を投入できる。

```
[ユーザー]: [発言内容]
```

投入された発言は次のラウンドで全ペルソナが参照する対象となる。

## ラウンド終了時の論点サマリー

各ラウンド終了後に必ず提示する。

```
📊 ラウンド N 論点サマリー

✅ 合意点:
  - [全ペルソナが同意している内容]
  - [議論で確認できた前提・制約]

❓ 未決定事項:
  - [まだ答えが出ていない重要な問い]
  - [ペルソナが提起したが深掘り不足の論点]

⚡ 対立点:
  - [ペルソナ間で意見が分かれている点と各立場]

---
続行しますか？

A) 続行（ラウンド N+1 へ）
B) 収束（成果物を出力する）
C) 特定の論点に絞って続行（論点を指定してください）
```

## 堂々巡り検出

同一の論点が2ラウンド以上継続した場合（進展がない場合）、以下を実行する:

1. 堂々巡りを明示する:

   ```
   ⚠️ 堂々巡り検出
   「[論点]」について N ラウンド議論が継続していますが、進展が見られません。
   ```

2. 新しい問いを立て直す:

   ```
   視点を変えて問い直します:
   「[従来の問い]」ではなく、「[新しい問い]」として考えた場合、どうなるでしょうか？
   ```

3. 新しい問いに対してラウンドを再開する

## ラウンド実行例

**機能意図**: 「ユーザーが広告枠を登録できる」

```
🔄 ラウンド 1 開始

**ビジネスアナリスト（固定）:**
UX デザイナーの「登録フォームはシンプルに」という主張に同意しつつも、
広告枠の「掲載期間」と「予算上限」の関係性が未定義のままです。
業界標準では CPM・CPC・CPV の課金モデルによってフィールドが大きく異なります。
課金モデルの決定なしに登録フォームを設計すると後から大規模な変更が必要になるリスクがあります。
根拠: IAB の広告規格（https://www.iab.com/guidelines/）

**UX デザイナー:**
ビジネスアナリストの課金モデル指摘は重要です。ただし、課金モデルを登録時に選択させると
認知負荷が高くなる懸念があります。Slack や Notion のオンボーディング事例では
「後から変更できる」設計で離脱率を下げています。デフォルト値＋後変更可能な設計を提案します。
根拠: Nielsen Norman Group のオンボーディング調査（2024）

...

📊 ラウンド 1 論点サマリー

✅ 合意点:
  - 登録フォームはシンプルであるべき
  - 課金モデルは後から変更できる設計が望ましい

❓ 未決定事項:
  - デフォルト課金モデル（CPM/CPC/CPV）をどれに設定するか
  - 掲載期間のバリデーション（最短・最長）はどう定義するか

⚡ 対立点:
  - 課金モデルを登録時に選択させるか（BA）vs デフォルト値で隠蔽するか（UX）

---
続行しますか？

A) 続行（ラウンド 2 へ）
B) 収束（成果物を出力する）※ ラウンド2以降のみ選択可
C) 特定の論点に絞って続行（論点を指定してください）
```

ユーザーが「A) 続行」を選択した場合:

```
🔄 ラウンド 2 開始

**ビジネスアナリスト（固定）:**
UX デザイナーの「デフォルト値＋後変更」案は実装観点から有効です。ただし、
デフォルトを CPM にすると動画広告に不向きで、後から CPV に変えた際の
過去データ整合性問題が発生します。登録時に「広告フォーマット（バナー/動画）」
を選択させ、フォーマットに応じて課金モデルを自動適用する設計を提案します。
根拠: Google Ads ヘルプセンター「課金モデルの選択」

**UX デザイナー:**
ビジネスアナリストの「フォーマット起点」アプローチは認知負荷を下げつつ
課金モデル問題を解決できる優れた案です。UX 観点でも「バナー/動画」の
2択は直感的です。追加で、掲載期間の最短を「1週間」に設定することで
スパム的な短期広告を防ぎ、PO・広告主双方に価値を提供できます。
根拠: 競合調査（Yahoo Ads・Meta Ads の最短掲載期間は1〜3日だが、
  本プロダクトは中小規模ターゲットであり1週間が業界慣行）

...

📊 ラウンド 2 論点サマリー

✅ 合意点:
  - 広告フォーマット（バナー/動画）を登録時に選択 → 課金モデル自動適用
  - 掲載期間の最短は1週間が妥当

❓ 未決定事項:
  - 動画フォーマットの場合の素材アップロード制限（サイズ・形式）

⚡ 対立点:
  - なし（ラウンド1の対立点は解消）

---
続行しますか？

A) 続行（ラウンド 3 へ）
B) 収束（成果物を出力する）
C) 特定の論点に絞って続行（論点を指定してください）
```

## 収束の基準（参考）

以下のいずれかを満たした場合に収束を提案できる（強制しない）:

- 主要な未決定事項に答えが出た
- 対立点について「どちらの方針を取るか」の判断材料が揃った
- ユーザーが「十分」と判断した
