# Step 0-2: 機能意図からのストーリー詳細化 — 詳細手順

## 入力

- Phase 定義書（`docs/requirements/PD-NNN-slug.md`）の「Epic マッピング」セクション
- 対象 Epic に対応する機能意図（1行〜数行の箇条書き）
- Inception 道具箱の成果物（任意）: `/aidd-mob`・`/aidd-inception-mock` 等の出力がある場合

## 手順

### 0-2-0. Inception 成果物の精緻化（成果物がある場合のみ）

Inception 道具箱（`/aidd-mob`・`/aidd-inception-mock`・`/aidd-inception-api-spec`・`/aidd-inception-cli`）の成果物がある場合、ゼロからストーリーを生成せず、成果物を「叩き台」として精緻化する（**収束フェーズ**）。

**精緻化の手順（4ステップ）:**

**① 重複検出・統合**: 意味的に同じストーリーを1つに統合する

- 同じユーザーアクションを異なる表現で記述しているものを統合
- UI 経由と API 経由の「同じ操作」は別ストーリーとして残す

**② 抜け補完**: 機能意図に対してカバーされていないシナリオを追加

- エラーケース・権限制御・空状態等が Inception 成果物に含まれていなければ補完
- 「As a...」形式で追加し「AI 補完」として区別

**③ 表現精鋭化**: 曖昧なストーリーを具体化する

- `[行動]` が「操作する」「管理する」等の抽象的な表現 → 具体的な動詞に変換
- `[理由]` がない → ビジネス目的を追記

**④ カバレッジ確認**: 全機能意図がいずれかのストーリーに対応しているか確認する

精緻化完了後、**0-2-3（ストーリー生成）をスキップして**ユーザーに確認を求める:

```
📝 Inception 成果物を精緻化しました

精緻化結果:
  統合: [N]件 → [M]件（重複[K]件統合）
  補完追加: [L]件

確定ストーリー一覧:
[精緻化後のストーリー一覧]

この内容でよいですか？
```

### 0-2-1. 機能意図の取得

Phase 定義書の「Epic マッピング」テーブルから対象 Epic に対応する機能意図を取得する。

```
例: 機能意図 = 「ユーザーが広告枠を登録できる」
```

### 0-2-2. ペルソナの特定

機能意図からペルソナを推定する。複数のペルソナが考えられる場合は全て列挙し、ユーザーに確認する。

```
機能意図「ユーザーが広告枠を登録できる」から以下のペルソナを推定しました:
- 広告主（広告枠を登録したい人）
- 管理者（広告主を管理したい人）

使用するペルソナはこれでよいですか？
```

### 0-2-3. ストーリーの生成

機能意図とペルソナを組み合わせて、「As a [ペルソナ], I want to [行動], so that [理由].」形式のストーリーを生成する。

**生成ルール:**

- 正常系のストーリーを中心に生成する
- 管理系・エラーケース・認可に関するストーリーも補完する
- 1ストーリー = 1つの明確なユーザーアクションに限定する
- 複合条件（「〇〇かつ△△できる」）は分割する

**提示フォーマット:**

```
機能意図「ユーザーが広告枠を登録できる」から以下のストーリーを生成しました:

### 人間の入力（機能意図から生成）
- S1: As a **広告主**, I want to 広告枠の基本情報（名称・形式・サイズ）を登録する, so that 配信先として使用できるようにするため.
- S2: As a **広告主**, I want to 登録した広告枠の内容を確認する, so that 間違いがないか検証できるようにするため.

### AI 補完（抜け漏れ補完）
- S3: As a **広告主**, I want to 広告枠の登録でエラーが発生した場合に原因を確認する, so that 修正して再登録できるようにするため.（AI 補完: エラーハンドリングは UX 上必須）
- S4: As a **管理者**, I want to 広告主が登録した広告枠を一覧で確認する, so that 不正な広告枠を検出・管理できるようにするため.（AI 補完: 管理者視点のストーリーが欠落）

変更・追加したいストーリーがあれば指示してください。
問題なければ「OK」で確定します。
```

### 0-2-4. ストーリーの確認・修正

ユーザーが確認・修正を行う対話フローを実行する。

| 操作 | 例 |
|------|-----|
| 追加 | 「S5: As a ... を追加」 |
| 削除 | 「S3 は不要」 |
| 修正 | 「S1 の so that を変えたい」 |
| 確定 | 「OK」「進めて」 |

### 0-2-5. /aidd-inception-mock プレースホルダー

> ⚠️ **Phase 13 以降で実装予定（#980）。** 現時点では以下のフローはスキップして Step 1 に進む。


## 完了時

以下の情報を Step 1 に渡す:

- 確定したストーリー一覧（S1〜Sn）
- 各ストーリーの出典区分（人間入力 / AI 補完）
- ペルソナ一覧
