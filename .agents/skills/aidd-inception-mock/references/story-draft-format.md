# ストーリードラフトフォーマット

画面モックから逆算したストーリードラフトの生成ガイド。`/aidd-new-epic` Step 0-2 の入力として使用できる形式で出力する。

## ストーリー生成ルール

> **共通ルール**: 粒度・ペルソナ推定・AI補完判定基準・情報源付記の詳細定義は `story-derivation-rules.md` を参照。
> 以下はこのファイル（定義元）のサマリー。

1. **1ストーリー = 1画面上の1操作** — 複合条件は分割する
2. **ペルソナを明示** — 機能意図から推定する
3. **正常系を中心に** — 管理系・エラー系も補完する
4. **根拠を付記** — どの画面・どの操作から導出したかを明記する

## 出力テンプレート

```markdown
## 🎨 inception-mock セッション成果物

対象機能意図: [機能意図]
生成画面数: N 画面
実施日: [yyyy-mm-dd]

---

## ストーリードラフト

> このセクションは /aidd-new-epic Step 0-2 の入力として使用します。

### 機能意図から導出したストーリー

- S1: As a **[ペルソナ]**, I want to [行動], so that [理由].
  根拠: [画面名] の [操作名]

- S2: As a **[ペルソナ]**, I want to [行動], so that [理由].
  根拠: [画面名] の [操作名]

### AI が補完提案したストーリー

- S3: As a **[ペルソナ]**, I want to [行動], so that [理由].
  根拠: [補完理由: エラーケース対応 / 権限制御 / 空状態対応 等]

- S4: As a **[ペルソナ]**, I want to [行動], so that [理由].
  根拠: [補完理由]

---

## 画面一覧（/aidd-new-epic 参照用）

| # | 画面名 | 種別 | 対応ストーリー |
|---|--------|------|--------------|
| 1 | [画面名] | [一覧/入力/詳細/etc.] | S1, S2 |
| 2 | [画面名] | [確認/完了] | S3 |
| 3 | [画面名] | [エラー] | S4 |

---

## 未決定事項

モック作成過程で浮かび上がった未決定事項。Epic の AC 定義前に解消が必要。

| # | 未決定事項 | 影響する画面 |
|---|-----------|------------|
| U1 | [未決定内容] | [画面名] |
| U2 | ... | ... |

---

## /aidd-new-epic への引き渡し方

/aidd-new-epic の Step 0-2 開始時に以下を提示する:

\`\`\`
inception-mock セッションの成果物があります。以下を Step 0-2 の入力として使用します:

[上記の成果物をここに貼付]

この内容をもとにストーリーを確認・補完し、AC を定義します。
\`\`\`
```

## /aidd-new-epic との対応関係

| inception-mock 成果物 | /aidd-new-epic での使用場所 |
|---------------------|--------------------------|
| ストーリードラフト（機能意図から） | Step 0-2: ストーリー詳細化の入力（AI 補完を省略） |
| ストーリードラフト（AI 補完分） | Step 0-2: 追加ストーリー候補として確認 |
| 画面一覧 | Step 1: AC 導出時の画面参照 |
| 未決定事項 | Epic 仕様書「未決定事項」セクションに転記 |
| 画面モック | G3（/aidd-screen-plan 等）の参考資料として活用 |
