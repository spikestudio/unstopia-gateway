# ローファイモックテンプレート

テキストベースのローファイモック生成ガイド。ASCII アートは使用しない。Markdown の構造で UI 要素を表現する。

## 基本フォーマット

```markdown
### [画面名]

> ユーザーはここで [主な目的・行動] を行う。

---

**[セクション名]**（例: ヘッダー / ナビゲーション / メインコンテンツ / フッター）

| 要素 | 種別 | 内容・状態 |
|------|------|-----------|
| ロゴ | 画像 | 左上配置 |
| ナビゲーション | リンク群 | [ページ名] / [ページ名] / ... |
| ユーザーメニュー | ドロップダウン | [ユーザー名] ▼ |

---

**メインコンテンツ**

[ タイトル: [画面タイトル] ]

[ 検索バー: プレースホルダー「[検索ヒント]」] [ 検索ボタン ]

[ フィルター: [項目1] / [項目2] / [項目3] ]

[テーブルの場合]
| [列名1] | [列名2] | [列名3] | 操作 |
|---------|---------|---------|------|
| [値] | [値] | [値] | [編集] [削除] |
| ... | ... | ... | ... |

[ ページネーション: ← 前 | 1 | 2 | 3 | 次 → ]

---

**アクションエリア**

[ [プライマリボタン] ]  [ [セカンダリボタン] ]
```

## 画面種別別テンプレート

### 一覧画面

```markdown
### [エンティティ名] 一覧

> ユーザーは登録済みの[エンティティ名]を一覧で確認し、絞り込み・個別操作を行う。

**ヘッダー**
[ タイトル: [エンティティ名] 管理 ]  [ + 新規登録 ]

**検索・フィルター**
[ 検索: [キーワード入力欄] ]  [ ステータス: すべて▼ ]  [ 検索ボタン ]

**一覧テーブル**
| [列1] | [列2] | [列3] | ステータス | 操作 |
|-------|-------|-------|-----------|------|
| [値] | [値] | [値] | [有効/無効] | [詳細] [編集] [削除] |

**空状態**（データが0件の場合）
[ アイコン ] まだ[エンティティ名]が登録されていません。
[ + 最初の[エンティティ名]を登録する ]

[ ページネーション: ← | 1 | 2 | → ]
```

### 登録・編集フォーム画面

```markdown
### [エンティティ名] [登録/編集]

> ユーザーは[エンティティ名]の情報を入力して保存する。

**フォームヘッダー**
[ タイトル: [エンティティ名]の[登録/編集] ]

**入力フィールド**
[フィールド名1] *必須
[ 入力欄: [プレースホルダー] ]

[フィールド名2]
[ 入力欄: [プレースホルダー] ]
ヒント: [入力ガイダンス]

[セレクト項目]
[ ドロップダウン: [選択してください ▼] ]

**バリデーション表示**（エラー時）
⚠️ [フィールド名1] は必須です。
⚠️ [フィールド名2] は[N]文字以内で入力してください。

**アクション**
[ 保存する ] [ キャンセル ]
```

### 詳細画面

```markdown
### [エンティティ名] 詳細

> ユーザーは選択した[エンティティ名]の詳細情報を確認し、関連操作を行う。

**ヘッダー**
← 一覧に戻る  |  [ 編集 ]  [ 削除 ]

**基本情報**
| 項目 | 値 |
|------|-----|
| [フィールド名1] | [値] |
| [フィールド名2] | [値] |
| ステータス | 🟢 有効 |
| 登録日時 | [yyyy-mm-dd HH:MM] |

**関連情報**
[サブセクション名]（N 件）
| [列1] | [列2] | 操作 |
|-------|-------|------|
| [値] | [値] | [詳細] |
```

### 確認・完了画面

```markdown
### 確認画面

> ユーザーは入力内容を最終確認してから送信する。

[ タイトル: 以下の内容で[アクション]しますか？ ]

**確認内容**
| 項目 | 内容 |
|------|------|
| [フィールド名1] | [入力値] |
| [フィールド名2] | [入力値] |

⚠️ この操作は取り消せません。

[ 確定する ] [ 修正する ]

---

### 完了画面

> 処理が完了したことをユーザーに伝える。

✅ [アクション名]が完了しました。

[ 一覧に戻る ]  [ 続けて登録する ]
```

### エラー画面

```markdown
### エラー画面

> システムエラーや権限エラーが発生した場合に表示する。

❌ エラーが発生しました

**エラーコード**: [コード]
**メッセージ**: [ユーザー向けの説明]

考えられる原因:
- [原因1]
- [原因2]

[ ページを再読み込みする ]  [ トップに戻る ]
```

## UX チェックリスト

モック作成後に以下を確認する:

| 観点 | チェック内容 |
|------|------------|
| ナビゲーション | 全画面で一貫したナビゲーションがあるか |
| 空状態 | データが0件の場合の表示があるか |
| エラー表示 | バリデーションエラーの表示方法が定義されているか |
| 確認ステップ | 破壊的操作（削除等）に確認ステップがあるか |
| 完了フィードバック | 操作完了後のフィードバックがあるか |
| 権限制御 | 権限によって異なる表示が考慮されているか |
