# ストーリー導出ルール（AC-E070-06）

全 Inception 子スキル（mock / api-spec / cli）共通のストーリー導出ルール。

## ストーリー粒度の定義

「1ストーリー = 1ユーザーアクション（目的単位）」

- 「UI から入力して保存する」と「API にリクエストを送信して保存する」は **同じユーザーアクション** → 1ストーリーに統合
- 「一覧を見る」と「一覧から絞り込む」は **別のアクション** → 2ストーリーに分離
- 複合条件（「〇〇かつ△△できる」）は **分割** する

## ペルソナ推定ルール

1. **機能意図のキーワードから推定**: 「管理者が」「一般ユーザーが」等の記述があれば使用
2. **画面/API/CLI の利用者から推定**: 管理画面 → 管理者、公開 API → 開発者/外部サービス
3. **複数ペルソナが考えられる場合**: 全て列挙してユーザーに確認

## AI 補完の判定基準

以下のカテゴリは、機能意図に明示的な記述がなくても AI が補完として提案してよい:

| カテゴリ | 補完条件 |
|---------|---------|
| エラーケース | 正常系のストーリーが存在する場合、対応するエラーストーリーを補完 |
| 空状態 | 一覧取得のストーリーがある場合 |
| 権限制御 | 複数ロールが想定される場合（管理者・一般ユーザー等） |
| 確認・完了 | 作成・更新・削除操作がある場合 |

## 情報源の付記ルール

全ストーリーに「どこから導出したか」を付記する:

```
- S1: As a **[ペルソナ]**, I want to [行動], so that [理由].
  根拠: [mock] 画面X の [操作名]
  根拠: [api-spec] [メソッド] /[パス]
  根拠: [cli] [コマンド] [サブコマンド]
  根拠: [AI補完] [補完理由（エラーケース対応 / 空状態対応 等）]
```

## ストーリーのフォーマット

```
As a **[ペルソナ]**, I want to [行動], so that [理由].
```

- `[行動]` は技術的な表現を避ける（「POST する」ではなく「登録する」）
- `[理由]` はビジネス上の目的を記述する
- ペルソナは **太字** で記載する

## /aidd-new-epic との関係

このルールで生成したストーリーは `/aidd-new-epic` Step 0-2 のインプットとして使用できる:

- AI によるストーリー補完（step0-story-elaboration.md の 0-2-3）を **省略可能**
- ただし「省略するかどうか」は `/aidd-new-epic` 内でユーザーが決定する
- Inception スキルの成果物はあくまでインプット資料であり、`/aidd-new-epic` が成果物を上書き・精緻化することを妨げない
