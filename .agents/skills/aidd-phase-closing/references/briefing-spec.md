# ブリーフィング仕様 — /aidd-phase-closing

`/aidd-phase-closing` は Step 6 で完了処理ブリーフィングを実行する。5 要素ブリーフィングを前提とし、Phase 完了処理固有の確認項目を追加定義する。

## 5 要素チェックリスト

ブリーフィングは以下の 5 要素を**全て**含むこと。1 つでも欠けている場合、ブリーフィングは不完全とみなす。

| # | 要素 | 内容 | 欠落時のリスク |
|---|------|------|-------------|
| 1 | 成果物の全内容の説明 | 完了した処理の全項目を漏れなく説明する。G6 記録・クローズ・更新・レトロスペクティブ結果を省略しない | 処理漏れに気づけない |
| 2 | AI が行った判断の根拠と代替案 | 「なぜこの手順を選んだか」「他にどんな対応があったか」を明示する | 判断の妥当性を評価できない |
| 3 | 暗黙の前提・行間の説明 | 完了処理に関わる前提（Milestone 名・Phase 番号等）を説明する | 前提の食い違いに気づけない |
| 4 | 未決定事項と論点の洗い出し | 次 Phase に持ち越した課題・未作成の改善 Issue を列挙する | 未対応項目が放置される |
| 5 | 人間に求める意思決定事項の明示 | 「次 Phase の開始準備をするか」「追加の改善 Issue があるか」を具体的に問いかける | 次アクションが曖昧になる |

## 品質基準

**「この説明だけ読めば Phase 完了処理の全体像と次のアクションが把握できる」品質を担保すること。**

具体的には:

- 完了した処理が全件チェックリスト形式で確認できる
- 作成した改善 Issue が一覧で確認できる
- 次 Phase への引き継ぎ事項が明確になっている
- 「はい/いいえ」で次アクションの承認判断できる程度に論点が整理されている

```markdown

### 完了処理サマリー

| 処理 | 結果 | 詳細 |
|------|------|------|
| G6 通過コメント記録 | ✅ 完了 | Epic #NNN・Phase Issue #NNN に記録 |
| Epic Issue クローズ | ✅ 完了 | N 件クローズ |
| Phase 定義書更新 | ✅ 完了 | `docs/requirements/PD-NNN-*.md` ステータス → 完了 |
| Milestone クローズ | ✅ 完了 | Phase N: [タイトル] |
| レトロスペクティブ | ✅ 完了 | 改善 Issue N 件作成 |

### レトロスペクティブ結果

#### Well（うまくいったこと）
- [項目]

#### Problem（問題・課題）
- [項目]

#### Try（作成した改善 Issue）
| # | Issue | タイトル | Milestone |
|---|-------|---------|-----------|
| 1 | #NNN | [タイトル] | Phase N+1 / 未割当 |

### 判断根拠と代替案

[処理過程で行った主要な判断。Milestone 特定方法・クローズ順序等]

### 暗黙の前提

[Milestone 名・Phase 番号・次 Phase の Milestone 存在有無等]

### 未決定事項・論点

[次 Phase の Milestone 未作成の場合の改善 Issue 割当・追加レトロスペクティブ項目等]

### AI 自信度

### 確認事項

- 上記の完了処理に漏れがないか確認してください
- 追加の改善 Issue が必要であれば教えてください
- 次 Phase の計画を開始しますか？（`/aidd-next` で確認）

Phase 完了処理が全て完了しました。`/aidd-next` で次のアクションを確認しますか？
```
