## 1. ビジョン

jinn をベースに、独自の AI gateway daemon として拡張していく。Antigravity エンジン対応・クロスセッション記憶システム・多層スキル/ツール管理を追加し、より強力な AI エージェント基盤を構築する。

## 2. ビジネスゴール

| # | ゴール | 測定指標 |
|---|--------|---------|
| 1 | 複数 AI エンジン (claude / codex / gemini / antigravity) を統一インターフェースで利用可能にする | 全エンジンで同一 API から呼び出し可能 |
| 2 | クロスセッション記憶により AI が過去の会話を参照できるようにする | セッション間での事実参照が可能 |
| 3 | 多層スキル管理 (Global → Dept → Agent) でエージェントごとのカスタマイズを実現する | スキル/ツールのスコープ制御が動作 |

## 4.1 スコープ外

- jinn オリジナルの機能への破壊的変更（アップストリームとのマージ容易性を保つ）
- 商用サービスとしての提供（個人利用・社内利用のみ）

## 5. 非機能要件（6 分類）

| 分類 | 定量目標（レンジ可） | 計測方法 | 優先度 |
|------|-------------------|---------|--------|
| **可用性** | ローカル daemon として常時稼働 | 手動確認 | SHOULD |
| **性能・拡張性** | エンジン呼び出しのオーバーヘッド 100ms 以内 | ログ計測 | SHOULD |
| **運用・保守性** | ホットリロード対応（設定変更に再起動不要） | 動作確認 | MUST |
| **移行性** | jinn からの移行スクリプト提供 | 移行テスト | COULD |
| **セキュリティ** | ローカル環境限定・外部公開なし | N/A | N/A |
| **環境・エコロジー** | N/A | N/A | N/A |
