## 対応ストーリー

- S13: As a **開発者**, I want to `packages/jimmy` のユニットテストを拡充し branch カバレッジを段階的に引き上げたい, so that E3 CI 閾値（最終 60%）を達成でき、変更の安全性を定量的に確認できる.
- S14: As a **開発者**, I want to テスタビリティを妨げるコードをリファクタリングしたい, so that テストが書きやすくなりカバレッジ拡充の効率が高まる.

## 概要

現状 branch カバレッジ 9.04%（ES-002 で計測確定）の `packages/jimmy` に対してユニットテストを拡充し、E3 CI 閾値（初期 10%、最終目標 60%）の段階的な引き上げを可能にする。

vitest 設定の修正（`include` パターン漏れ・テストファイルの `coverage.exclude` 漏れ）、テストファイルの配置統一、テストが書きにくいコードのリファクタリングも合わせて実施する。

## バリデーションルール

| フィールド | ルール | エラー時の振る舞い |
|-----------|--------|------------------|
| vitest `include` パターン | `src/**/__tests__/**/*.test.ts` に全テストファイルが含まれること | パターン外のテストファイルは `__tests__/` に移動する |
| カバレッジ閾値 | branch 30%（中間目標）→ 60%（最終目標） | 閾値未達の場合は追加テストを実装する |

## ステータス遷移（該当なし）

テスト追加・リファクタリングのみ。エンティティのステータス遷移なし。

## エラーケース

| ケース | 条件 | 期待する振る舞い | 説明 |
|--------|------|----------------|------|
| テストが `__tests__/` 外に存在する | `src/**/*.test.ts` が `__tests__/` 外に配置されている | `__tests__/` への移動を実施し vitest include パターンに合わせる | 現状 `queue.test.ts` 等 3 ファイルが該当 |
| カバレッジ閾値未達 | branch カバレッジが目標値未満 | 追加テストを実装して閾値を達成する | 60% 達成困難な場合は E3 で段階的閾値を設定 |
| リファクタリングによるリグレッション | 既存テストが FAIL する | `pnpm test` が PASS するまで修正する | AC-E003-07 で検証 |

## 非機能要件

| 項目 | 基準 |
|------|------|
| テスト実行時間 | カバレッジ計測を有効にした `pnpm test --coverage` が 120 秒以内に完了する |
| テストの独立性 | 各テストは他のテストに依存せず、任意の順序で実行できる |
| モック方針 | エンジン・ファイルシステム・外部 CLI への依存はモックで代替する（`docs/conventions/testing.md` 準拠） |
