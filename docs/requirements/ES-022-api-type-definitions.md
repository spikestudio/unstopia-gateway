## 対応ストーリー

- S17: As a **開発者**, I want to API リクエスト・レスポンスの型定義を整備したい, so that `api.ts` 分割時に型の参照が一箇所になり、変更漏れを防げる.

## 概要

`gateway/api/` 配下の各ハンドラー（sessions・org・cron・connectors・skills・misc・stt）では、リクエストボディのパースに `body as Record<string, unknown>` キャストが多用されており、レスポンス形状も `json(res, { ... })` に匿名オブジェクトとして直接記述されている。型の定義が分散・暗黙的なため、エンドポイントを変更するたびに関連する型の変更漏れが生じやすい。

このEpicでは `gateway/api/api-types.ts` を新設し、各エンドポイントのリクエスト・レスポンス型を集約する。既存ハンドラーは新型定義を参照するよう更新する。型安全性の向上と変更漏れ防止が目的であり、ランタイム動作の変更は行わない。
