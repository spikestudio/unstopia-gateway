## 対応ストーリー

<!-- Phase 定義書 PD-001 §3 から転記 -->

- S6: As a **開発者**, I want to テストカバレッジを計測できるようにしたい, so that 変更の安全性を定量的に確認できる.
- S7: As a **開発者**, I want to E2E テストの整備状況を把握・改善したい, so that 主要なユーザーフローが自動検証されている状態を作れる.

## 概要

`packages/jimmy` に `@vitest/coverage-v8` を追加してカバレッジ計測を有効にする。`pnpm test --coverage` で branch カバレッジを計測し、現状値（9.04%）を把握した。E3 CI 初期閾値は **10%** に確定済み（テスト拡充後に段階引き上げ予定）。また `e2e/smoke.spec.ts` の現状把握を実施し、E2E 整備方針は**現状維持**に確定した。

## ストーリーと受入基準

### Story 6: テストカバレッジ計測基盤の整備

> As a **開発者**, I want to テストカバレッジを計測できるようにしたい, so that 変更の安全性を定量的に確認できる.

**受入基準:**

- [x] **AC-E002-01**: 開発者が `pnpm test --coverage` を `packages/jimmy` ディレクトリで実行すると、カバレッジレポートが生成され、ターミナルに branch / statement / line / function の各カバレッジ率が表示される。 ← S6
- [x] **AC-E002-02**: 生成されたカバレッジレポートが `packages/jimmy/coverage/` に HTML 形式でも出力され、ブラウザで参照できる。 ← S6
- [x] **AC-E002-03**: カバレッジ計測対象が `packages/jimmy/src/**/*.ts` 全体（テストファイル・型定義ファイルを除く）を網羅しており、計測漏れのモジュールが存在しない。 ← S6
- [x] **AC-E002-04**: `pnpm test --coverage` の実行後、branch カバレッジの現状値が計測され、目標閾値（E3 初期: 10%、テスト拡充後に段階引き上げ予定）が確定している。（AI 補完: 閾値設定の前に現状値を把握しないと適切な目標が設定できないため追加）← S6
- [x] **AC-E002-05**: `vitest.config.ts` に `coverage` 設定が追記されており、`provider: 'v8'`・`reporter: ['text', 'html']`・`include: ['src/**/*.ts']`・`exclude: ['src/**/*.test.ts', 'src/**/__tests__/**']` が設定されている。 ← S6

### Story 7: E2E テスト整備状況の把握と補完

> As a **開発者**, I want to E2E テストの整備状況を把握・改善したい, so that 主要なユーザーフローが自動検証されている状態を作れる.

**受入基準:**

- [x] **AC-E002-06**: `e2e/smoke.spec.ts` の現在のテスト内容（カバーするシナリオ・前提条件・実行可否）が把握され、ドキュメントに記録されている。 ← S7
- [x] **AC-E002-07**: 既存の `e2e/smoke.spec.ts` が `pnpm test:e2e`（playwright）で正常に実行でき（gateway が起動している環境前提）、PASS または SKIP（環境依存で実行不可）の状態が確認されている。 ← S7
- [x] **AC-E002-08**: E2E テストの整備方針（「現状維持」または「追加テストケース実装」）がユーザーと合意されており、方針が Epic 仕様書の未決定事項セクションに記録されている。（AI 補完: E2E は環境依存が高く、方針を先に確定しないと無駄な実装が発生するリスクがあるため追加）← S7

**インターフェース:** カバレッジレポート（HTML / テキスト出力）。外部 API なし。

| coverage.threshold（branch） | 0〜100 の数値。未設定時はデフォルト（閾値強制なし） | vitest が設定エラーをターミナルに出力し、テストを中断する |

## ステータス遷移（該当する場合）

該当なし（ツール設定変更のみ）

## エラーケース

| ケース | 条件 | 期待する振る舞い | 説明 |
|--------|------|----------------|------|
| @vitest/coverage-v8 未インストール | `coverage` 設定があるが依存が未インストール | vitest が `Missing coverage provider` エラーを出力し終了コード 1 で停止 | インストール手順が README に記載されている |
| カバレッジ閾値未達 | branch カバレッジが設定閾値を下回る | vitest がカバレッジ閾値エラーを出力し終了コード 1 で停止 | CI でこの挙動を利用してゲートとする |
| テストファイルが `include` パターン外 | `src/**/*.ts` に含まれないが計測対象にしたいファイルがある | カバレッジレポートにそのファイルが含まれない | `include` 設定の見直しが必要 |

## 非機能要件

| 項目 | 基準 |
|------|------|
| テスト実行時間 | カバレッジ計測を有効にした状態での `vitest run` が 60 秒以内に完了する |
| カバレッジレポートサイズ | `coverage/` ディレクトリを `.gitignore` に追加し、リポジトリに含めない |
| ツール選定 | @vitest/coverage-v8（PD-001 §13 で採用確定。istanbul との比較で v8 が Vitest 公式推奨） |
