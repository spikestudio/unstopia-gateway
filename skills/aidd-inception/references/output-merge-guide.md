# サブスキル成果物統合ガイド（AC-E070-04）

複数サブスキルを実行した場合の成果物収集・ストーリー統合・重複排除の手順。

## 成果物収集手順

各サブスキル実行完了後、以下を即座に収集してメモリ上に保持する:

```
[サブスキル名] 完了 — 収集:
  stories: ["S1: As a ...", "S2: As a ..."]
  artifacts: {type: "画面一覧 / API一覧 / CLI一覧", content: "..."}
  open_questions: ["Q1: ...", "Q2: ..."]
  sources: ["URL1", "文書名1"]
```

## 2つ目以降のサブスキルへの引き渡し

2つ目のサブスキルを起動する際、1つ目の成果物を引き渡す:

```
Agent(prompt="/aidd-inception-api-spec を実行してください。

機能意図: [機能意図]

前回の /aidd-inception-mock セッションで以下が確認済みです:
画面構成: [画面名一覧]
確定したペルソナ: [ペルソナ名一覧]
リサーチ情報源: [URL一覧]

これらを踏まえて API 仕様を設計してください。")
```

## ストーリー重複排除ルール（AC-E070-04）

### 重複の判定基準

| ケース | 判定 | 処理 |
|--------|------|------|
| 同じペルソナが同じ目的を記述している | 重複 | 詳細な方を残す |
| UI 経由の操作 vs API 経由の同操作 | 重複しない | 両方残す（実装パスが異なる） |
| 同じ操作だが詳細度が異なる | 重複 | 詳細な方を残す |
| ペルソナは違うが目的が同一 | 重複 | 統合して「複数ペルソナが対象」と記載 |

### 統合手順

1. 全サブスキルのストーリーを仮番号で列挙（M1, M2... / A1, A2... / C1, C2...）
2. 上記ルールで重複チェック
3. 重複排除後に通し番号（S1, S2...）を付け直す

### 統合版ストーリードラフトの出力フォーマット

```markdown
## ストーリードラフト（統合版）

統合前: [mock]N件 + [api-spec]M件 → 重複K件統合 → 統合後: (N+M-K)件

### 機能意図から導出したストーリー
- S1: As a **[ペルソナ]**, I want to [行動], so that [理由].
  根拠: [mock] 画面X / [api-spec] GET /xxx（両スキルで確認）

### AI が補完提案したストーリー
- S2: As a **[ペルソナ]**, I want to [行動], so that [理由].
  根拠: [補完理由]

### 未決定事項（統合・重複排除済み）
- Q1: [問い] → 影響: S1, S3
```
