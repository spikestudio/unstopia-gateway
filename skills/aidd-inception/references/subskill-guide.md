# サブスキル選定・委譲ガイド

`/aidd-inception` 親スキルがサブスキルを選定・委譲するための詳細ガイド。

## サブスキル一覧

| サブスキル | 対象プロジェクト | 主な成果物 | 実装スキル |
|-----------|----------------|----------|----------|
| `/aidd-inception-mock` | UI系（Web・モバイル・デスクトップ） | 画面モック + ストーリードラフト | `skills/aidd-inception-mock/SKILL.md` |
| `/aidd-inception-api-spec` | API系（REST・GraphQL・gRPC） | API仕様 + ストーリードラフト | `skills/aidd-inception-api-spec/SKILL.md` |
| `/aidd-inception-cli` | CLI系（コマンドラインツール） | CLI仕様 + ストーリードラフト | `skills/aidd-inception-cli/SKILL.md` |

## サブスキル自動推定ロジック

以下の優先順序でプロジェクト特性を判定する:

1. **引数指定**: `mock` / `api-spec` / `cli` が指定されている場合はそれを使用
2. **CLAUDE.md 参照**: `project_type:` の記述があればそれを参照
3. **機能意図分析**: キーワードから推定

| キーワード（機能意図に含まれる場合） | 推定サブスキル |
|----------------------------------|-------------|
| 画面・UI・フォーム・表示・一覧・管理画面 | mock |
| API・エンドポイント・バックエンド・サーバー・データ | api-spec |
| CLI・コマンド・ターゲット・ターミナル・シェル | cli |
| 複数キーワード混在 | 複数サブスキル |

## サブスキル委譲の実装

### 単一サブスキルの委譲

```
Agent(prompt="/aidd-inception-mock を実行してください。

機能意図: [機能意図]
参考情報（リサーチ結果）: [リサーチサマリー]
")
```

### 複数サブスキルの順次委譲

UI + API の複合プロジェクトの場合:

```
# Step 1: 画面モックを生成
Agent(prompt="/aidd-inception-mock を実行してください。機能意図: [機能意図]")

# Step 2: API 仕様を生成（モック結果も参照）
Agent(prompt="/aidd-inception-api-spec を実行してください。機能意図: [機能意図]。
なお、画面モックセッションで以下の画面構成が確定しています: [モック結果のサマリー]")
```

## 複数サブスキル実行時の統合方法

複数のサブスキルを実行した場合、ストーリードラフトを統合する:

1. 各サブスキルのストーリードラフトを収集
2. 重複するストーリーを統合（同じユーザーアクションを指すものは1つにまとめる）
3. 統合版ストーリードラフトをユーザーに確認
4. 承認後に `/aidd-new-epic` へ引き渡し
