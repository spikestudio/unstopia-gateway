# /aidd-inception — Inception 道具箱オーケストレーター

G1 後・`/aidd-new-epic` 実行前に任意で使用する親スキル。プロジェクト特性を分析してサブスキル（`/aidd-inception-mock`・`/aidd-inception-api-spec`・`/aidd-inception-cli`）を選択・委譲し、具体的な成果物とストーリードラフトを生成して `/aidd-new-epic` に引き渡す。

## インターフェース

### 前提条件

| 前提 | 確認方法 | 未完了時の案内 |
|------|---------|-------------|
| G1 通過済み（Phase 定義書が承認済み） | `docs/requirements/PD-*.md` にステータス「承認済み」が存在する | `/aidd-new-phase` を先に実行 |

### capability 呼び出し（委譲先）

| サブスキル | 起動条件 | 委譲方法 |
|-----------|---------|---------|
| `/aidd-inception-mock` | UI 系プロジェクト・画面モックが必要 | `Agent(prompt="/aidd-inception-mock を実行してください。機能意図: [X]")` |
| `/aidd-inception-api-spec` | API 系プロジェクト・バックエンドファースト | `Agent(prompt="/aidd-inception-api-spec を実行してください。機能意図: [X]")` |
| `/aidd-inception-cli` | CLI ツール系プロジェクト | `Agent(prompt="/aidd-inception-cli を実行してください。機能意図: [X]")` |
| `/aidd-research` | Step 2（プロジェクト特性リサーチ） | 直接呼び出し |

### 入力

| 項目 | 形式 | 必須 | 説明 |
|------|------|------|------|
| 機能意図 または 機能名 | 引数 | 任意 | 省略時はユーザーに確認 |
| サブスキル指定 | 引数 | 任意 | `mock` / `api-spec` / `cli`。未指定時は自動推定 |

### 出力

| 項目 | 形式 | 説明 |
|------|------|------|
| 具体的成果物 | Markdown | 選択したサブスキルの成果物（モック / API仕様 / CLI仕様） |
| ストーリードラフト | Markdown | 成果物から逆算した主要ストーリー一覧 |
| 引き渡しメモ | Markdown | `/aidd-new-epic` Step 0 インプット互換フォーマット |

### 後続スキル

| スキル | 条件 |
|--------|------|
| `/aidd-new-epic` | 成果物・ストーリードラフト完成後、引き渡して Epic 定義に進む |

## Step 0: 入力受付

1. 機能意図が未指定の場合はユーザーに確認する:

   ```
   /aidd-inception を開始しますか？
   - 対象機能意図または機能名を入力してください
   ```

2. `/aidd-mob` を先に実行した場合、その成果物を受け取る（AC-E070-09）:

   ```
   /aidd-mob の成果物がありますか？（あればここに貼り付けてください）
   → mob 成果物がある場合: ストーリードラフトをサブスキルへの引き渡し情報として使用します
   → mob 成果物がない場合: サブスキルが自律リサーチで代替します
   ```

3. Phase 定義書から対象機能意図を取得し、セッション開始を宣言する:

   ```
   🚀 inception セッション開始
   対象機能意図: [機能意図]

   プロジェクト特性を分析してサブスキルを選択します...
   ```

## Step 1: サブスキル選定

**AC-E068-01: サブスキル自動推定:**

機能意図・CLAUDE.md・プロジェクト構成から以下を分析してサブスキルを推定する:

| プロジェクト特性 | 推奨サブスキル |
|----------------|-------------|
| 画面・UI が主体 | `/aidd-inception-mock` |
| API・バックエンドが主体 | `/aidd-inception-api-spec` |
| CLI ツールが主体 | `/aidd-inception-cli` |
| 複合（UI + API 等） | 複数サブスキルを順次実行 |

**AC-E068-02: ユーザー承認フロー:**

推定結果をユーザーに提示し承認を得る:

```
📋 サブスキル選択案

プロジェクト特性: [分析結果]
推奨サブスキル: /aidd-inception-[種別]
理由: [選択理由]

この選択でよいですか？変更する場合はサブスキルを指定してください。
（mock / api-spec / cli / mock+api-spec 等の組み合わせも可能）
```

承認なしにサブスキルを起動しない。

## Step 2: AI リサーチ・補完

**AC-E068-03: プロジェクト特性リサーチ:**

`/aidd-research` を呼び出して機能意図に関連する情報を収集する:

- 類似プロダクトの設計パターン
- 見落とされがちな要件・エッジケース
- 業界固有の制約・ベストプラクティス

リサーチ結果をサブスキルへの引き渡し情報に含める。

## Step 3: サブスキル委譲・実行

**AC-E068-04: サブスキル実行:**

承認されたサブスキルを `Agent(prompt=...)` 形式で委譲する。

複数サブスキルを実行する場合は **順次実行**（1つ目の成果物を2つ目に引き渡す）。
引き渡し方法は `references/output-merge-guide.md` の「2つ目以降のサブスキルへの引き渡し」を参照。

```
🔄 /aidd-inception-[種別1] を実行中...
✅ /aidd-inception-[種別1] 完了 — 成果物を収集しました
🔄 /aidd-inception-[種別2] を実行中...（[種別1]の成果物を引き渡し済み）
```

実行完了後、各成果物を収集して Step 4 に進む。

**サブスキル失敗時のフォールバック（S6）:**

1. **リトライ**: 同じプロンプトで1回再実行する
2. **インライン実行**: リトライも失敗した場合、親スキルがサブスキルの処理をインラインで実行する（サブスキルの SKILL.md を参照して同等の処理を直接実施）
3. ユーザーへの丸投げはしない

## Step 4: ストーリードラフト統合・確認

**AC-E068-05: 成果物統合とユーザー確認:**

`references/output-merge-guide.md` の「ストーリー重複排除ルール」に従って統合し、ユーザーに確認する。

各サブスキルの成果物（ストーリードラフト含む）を統合し、ユーザーに確認する:

```
✅ inception セッション完了

生成した成果物:
- [サブスキル名]: [成果物の概要]

ストーリードラフト（統合版）:
- S1: ...
- S2: ...
...

この内容で /aidd-new-epic に引き渡してよいですか？
```

ユーザーの承認後、Step 5 に進む。

## Step 5: 引き渡し準備

**AC-E068-08: /aidd-new-epic 互換フォーマットで出力:**

全成果物を `/aidd-new-epic` Step 0 インプット互換フォーマットで出力する。

出力後:

```
🎉 inception セッション完了

成果物を /aidd-new-epic に引き渡す準備ができました。

次のステップ:
→ /aidd-new-epic [Epic名] を実行してください。
→ Step 0（ストーリー詳細化）で上記の成果物を参照します。
```

## 重要なルール

- **発散フェーズ**: このスキルは発散フェーズに属する。アイデア・ストーリー候補・制約・対立点を積極的に増やすことが目的。成果物は「叩き台」であり、`/aidd-new-epic` の収束フェーズで精緻化される
- 日本語で対話する
- **FRAMEWORK.md の意思決定基準に従う**
- **ユーザー確認なしの自動実行禁止** — サブスキル選定・各サブスキルの実行前に必ず承認を得る
- **サブスキルは `Agent(prompt=...)` 形式で委譲する** — `Agent(skill=...)` という形式は存在しない
- **情報源付記必須** — リサーチ結果には必ず参照元を付記する
- ブリーフィング省略禁止
