> **強度の定義:**
>
> - **MUST**: 必ず従う。違反はレビューで MUST FIX
> - **SHOULD**: 原則従う。例外にはコメントで理由を記載
> - **MAY**: 推奨。判断で採否を決めてよい

## 1. コントラスト・アクセシビリティ

<!-- 要件: WCAG 2.1 AA 準拠の基準を定義する -->

| # | ルール | 強度 | 根拠 |
|---|--------|------|------|
| A-01 | 通常テキスト（24px 未満）は背景に対して **4.5:1 以上** | MUST | WCAG 2.1 AA 1.4.3 |
| A-02 | 大きなテキスト（24px 以上 / 18.5px 太字以上）は **3:1 以上** | MUST | WCAG 2.1 AA 1.4.3 |
| A-03 | UI コンポーネント（アイコン・ボーダー）は隣接色に対して **3:1 以上** | MUST | WCAG 2.1 AA 1.4.11 |
| A-04 | タッチターゲットは最低 **44x44 CSS px** | SHOULD | WCAG 2.5.5 |
| A-05 | フォーカスリングは **2px 以上**、背景に対して 3:1 以上 | MUST | WCAG 2.4.7 |
| A-06 | フォーカスは `:focus-visible` で適用する | SHOULD | |
| A-07 | 色のみに依存して情報を伝えない。アイコン・テキストを併用する | MUST | WCAG 1.4.1 |

## 2. インタラクティブ状態

| # | ルール | 強度 |
|---|--------|------|
| I-01 | ボタンは **default / hover / active / focus / disabled** の 5 状態を全て定義する | MUST |
| I-02 | hover は同系色の明度シフトで表現する | SHOULD |
| I-03 | disabled は `opacity: 0.4` + `cursor: not-allowed` | MUST |
| I-04 | disabled 要素に hover / active スタイルを適用しない | MUST |
| I-05 | 状態遷移のトランジションは **150-200ms, ease-out** | SHOULD |

## 3. フォーム

| # | ルール | 強度 |
|---|--------|------|
| F-01 | フォームラベルはフィールドの **上** に配置する | MUST |
| F-02 | エラーメッセージはエラーフィールドの **直下にインライン** 表示する | MUST |
| F-03 | エラーは `destructive` 色 + アイコンの組み合わせで表示する | MUST |
| F-04 | エラーは「**原因**」+「**解決策**」の 2 要素を含む | MUST |
| F-05 | バリデーションは `onBlur` で実行する | SHOULD |

## 4. ローディング・空状態・エラー状態

### ローディング

| # | ルール | 強度 |
|---|--------|------|
| L-01 | 200ms 以上のデータ取得には **スケルトン UI** を表示する | MUST |
| L-02 | スケルトンは最終コンテンツと **同じサイズ** を予約する | MUST |
| L-03 | スケルトンに `aria-busy="true"` + `role="status"` を設定する | MUST |

### 空状態

| # | ルール | 強度 |
|---|--------|------|
| EM-01 | 空状態は **見出し + 説明文 + CTA** の 3 要素で構成する | MUST |
| EM-02 | CTA はコンテキストに応じた **具体的アクションラベル** にする | MUST |

### エラー状態

| # | ルール | 強度 |
|---|--------|------|
| ER-01 | エラーメッセージは「原因」+「解決策」を含む | MUST |
| ER-02 | エラー Toast は **自動消去しない** | MUST |
| ER-03 | 重大度に応じて表示パターンを使い分ける（フィールド→インライン、ページ→バナー、致命的→モーダル） | MUST |

## 5. レスポンシブ

| # | ルール | 強度 |
|---|--------|------|
| RE-01 | **mobile-first**（`min-width`）で実装する | MUST |
| RE-02 | モバイルのタッチターゲットは最低 **48x48dp** | SHOULD |
| RE-03 | レイアウトは **相対単位**（rem, %, ch, vw）を優先する | SHOULD |
