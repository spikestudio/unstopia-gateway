# カテゴリ: branch-protection（ブランチ保護ルール）

GitHub リポジトリのブランチ保護ルール（Branch Protection Rules）を対話的に設定する。

## 前提条件

- GitHub リポジトリが存在する（`.git/` + `gh repo view`）
- gh CLI が認証済み（`gh auth status`）
- リポジトリの Admin 権限を持っている

## 手順

### 1. 現在の設定を検出

対象リポジトリとデフォルトブランチの情報を取得する:

```bash
# リポジトリ情報
gh repo view --json name,owner,defaultBranchRef

# 既存のブランチ保護ルール
gh api repos/{owner}/{repo}/branches/{default_branch}/protection
```

結果を一覧で表示する:

| 設定項目 | 現在の状態 |
|---------|----------|
| PR 必須 | ✅ / ❌ |
| レビュー必須数 | N 人 / ❌ 未設定 |
| stale レビュー却下 | ✅ / ❌ |
| CI ステータスチェック必須 | ✅ / ❌ |
| 必須チェック名 | — |
| 最新ブランチ必須 | ✅ / ❌ |
| 管理者にも適用 | ✅ / ❌ |
| force push 禁止 | ✅ / ❌ |
| ブランチ削除禁止 | ✅ / ❌ |

### 2. チーム構成の確認

```
チーム構成を教えてください:
1. 1人チーム（ソロ開発）
2. 2-3人チーム
3. 4人以上のチーム
```

### 3. 推奨設定の提示とカスタマイズ

チーム構成別の推奨設定:

| 設定項目 | 1人チーム | 2-3人チーム | 4人以上 |
|---------|----------|------------|--------|
| PR 必須 | ✅ | ✅ | ✅ |
| レビュー必須数 | 0 | 1 | 2 |
| stale レビュー却下 | — | ✅ | ✅ |
| CI ステータスチェック | ✅ | ✅ | ✅ |
| 管理者にも適用 | ✅ | ✅ | ✅ |
| force push 禁止 | ✅ | ✅ | ✅ |
| ブランチ削除禁止 | ✅ | ✅ | ✅ |

推奨設定をユーザーに提示し、カスタマイズを受け付ける。

### 4. 設定の適用

1. CI ステータスチェック名を特定（gh api で直近のチェック名を取得）
2. 必須ステータスチェックに `Merge Gate Check`（merge-guard.yml のジョブ）を追加する
3. 確定した設定を GitHub API で適用（`gh api --method PUT`）
4. 適用後に設定を再取得して検証

**注意:**

- API 呼び出しの前に必ずユーザーの承認を得る
- 既存の保護ルールがある場合は上書きの確認を取る
- CI ステータスチェック名が不明な場合は無理に設定せず、スキップを案内
- gh CLI の認証エラーや権限不足の場合は明確なエラーメッセージと対処法を案内

## 完了条件

- デフォルトブランチに保護ルールが設定されている
- PR 必須化 + CI ステータスチェックが有効
- force push / ブランチ削除が禁止されている
- 管理者にも適用が有効
- 適用後の設定再取得で検証が完了している
- 緊急時の無効化方法が案内されている（リポジトリ Settings → Branches）
