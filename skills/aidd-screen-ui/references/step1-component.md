# Step 1: UIKit コンポーネント選定 — 詳細手順

## 1-1. UI 要素の抽出

各 SCR-ID の画面仕様書（`docs/design/screen-spec-[slug]/[SCR-ID]-*.md`）を読み込み、「コンポーネント選定」セクションまたはフィールド定義から UI 要素を列挙する。

| 画面種別 | 典型的な UI 要素 |
|---------|----------------|
| フォーム | 入力フィールド・セレクト・チェックボックス・ラジオ・送信ボタン・キャンセルボタン |
| 一覧 | テーブル・ページネーション・検索バー・フィルター・ソート・アクションボタン |
| 詳細 | 読み取り専用フィールド・編集ボタン・削除ボタン・タブ |
| モーダル | ダイアログ・確認ボタン・キャンセルボタン |
| ダッシュボード | 統計カード・グラフ・テーブル・ヘッダー |

## 1-2. catalog.json からの候補抽出

各 UI 要素に対して、catalog.json の `templates` 配列を以下の基準でフィルタリングし候補を生成する:

| フィルタリング基準 | 方法 |
|-----------------|------|
| 画面種別との照合 | `category`・`layer` フィールドで絞り込む（`ui`→基本コンポーネント、`composite`→複合コンポーネント） |
| UI 要素名との照合 | `id`・`tags` フィールドで関連コンポーネントを抽出 |
| `useWhen` の確認 | 「この UI 要素の用途に合っているか」を確認 |

## 1-3. 候補提示フォーマット

各 UI 要素に対して以下の形式で候補を提示する:

```
## [SCR-ID]: [画面名] — コンポーネント選定

### UI 要素: [要素名]

推奨候補:
1. **[コンポーネント名]**（catalog ID: [id]）
   - useWhen: [useWhen の内容]
   - インポート: `import { X } from '@spikestudio/uikit/components/...'`
2. **[コンポーネント名]**（catalog ID: [id]）
   - ...
3. **カスタム実装**（UIKit に適切なコンポーネントがない場合）

どれを選びますか？（1 / 2 / 3 / 別のコンポーネント名を入力）
```

## 1-4. 選択・変更の受け付け

ユーザーからの選択指示を受け付ける:

- 番号を入力 → 対応するコンポーネントを選定として記録する
- コンポーネント名を直接入力 → catalog.json で検索し、見つかれば詳細を提示してから確定する
- 「カスタム実装」または「3」→ 「カスタム実装」として記録し次の要素へ進む
- 「全て推奨で OK」→ 各要素の推奨 1 番目を一括採用し、一覧提示して確認を求める

## 1-5. UIKit 未対応要素の処理

catalog.json に適切なコンポーネントが見つからない場合（エラーで停止しない）:

```
[要素名] に対応する UIKit コンポーネントが見つかりませんでした。

「カスタム実装」として記録します。実装時に適切なコンポーネントを選定してください。
続行しますか？（OK）
```

## 1-6. 選定結果の確認

全 UI 要素の選定が完了したら、選定結果の一覧を提示してユーザーの承認を求める:

```
## コンポーネント選定結果: [SCR-ID] [画面名]

| UI 要素 | 選定コンポーネント | インポートパス |
|---------|----------------|-------------|
| [要素名] | [コンポーネント名] | `import { X } from '...'` |
| [要素名] | カスタム実装 | — |

この選定で進めてよいですか？（OK / 変更したい要素を指定）
```

## 完了時

全 SCR-ID の全 UI 要素のコンポーネント選定が完了したら、選定結果を内部に記録して Step 2 に進む。
