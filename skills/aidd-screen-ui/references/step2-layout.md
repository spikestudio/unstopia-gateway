# Step 2: レスポンシブレイアウト設計 — 詳細手順

## ブレークポイント定義

| ブレークポイント | 幅 | 対象デバイス |
|--------------|-----|------------|
| sm | < 768px | スマートフォン |
| md | 768px〜1024px | タブレット |
| lg | > 1024px | デスクトップ |

## 2-1. シンプル画面の省略判定

以下の条件に該当する画面は「レスポンシブ対応不要」として省略できる。ユーザーに確認を求めてから省略する:

| 条件 | 例 |
|------|-----|
| 画面種別がモーダル・ダイアログ | 削除確認ダイアログ |
| 全ブレークポイントで同一レイアウト | シンプルなエラー画面 |
| ユーザーが「省略で OK」と指示 | — |

省略時のフォーマット:

```
レスポンシブ対応不要: [理由]（例: モーダル画面のため全幅固定）
```

## 2-2. レイアウト構成の定義

各ブレークポイントでのレイアウトを対話的に定義する:

```
## [SCR-ID]: [画面名] — レスポンシブレイアウト設計

### sm（モバイル）のレイアウト案:
- 1カラム縦スタック
- ナビゲーションはハンバーガーメニュー
- ボタンはフル幅

この案でよいですか？（OK / 変更内容を入力）
```

## 2-3. 各ブレークポイントの定義項目

| 定義項目 | 説明 |
|---------|------|
| カラム数 | グリッドのカラム分割数（1〜12） |
| 表示/非表示 | ブレークポイントで非表示になる要素 |
| スタック方向 | 横並び（row）か縦積み（column）か |
| コンポーネントサイズ | `size` prop の変化（sm→lg: small→default など） |
| ナビゲーション形式 | サイドバー/トップナビ/ハンバーガーメニュー |

## 2-4. レイアウト確認

定義完了後にブレークポイント差分テーブルを提示してユーザーに確認を求める:

```
## レイアウト差分: [SCR-ID] [画面名]

| 要素 | sm | md | lg |
|------|-----|-----|-----|
| レイアウト | 1カラム | 2カラム | 3カラム |
| サイドバー | 非表示 | 表示 | 表示 |
| ボタン幅 | フル幅 | 自動 | 自動 |

この設計で進めてよいですか？（OK / 変更）
```

## 完了時

全 SCR-ID のレイアウト設計が完了したら内部に記録して Step 3 に進む。
