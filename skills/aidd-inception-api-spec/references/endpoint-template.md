# API エンドポイントテンプレート

リクエスト/レスポンス例の生成ガイド。JSON 形式を基本とし、Markdown で構造化する。

## エンドポイント仕様フォーマット

各エンドポイントは以下の構造で記述する:

- **概要**: エンドポイントの目的を1行で
- **認証**: 必須 / 不要
- **リクエストパラメータ**: パス/クエリ/ボディのパラメータ一覧と型・必須・説明
- **リクエストボディ例**: POST/PUT/PATCH のみ JSON 形式
- **レスポンス例**: 成功時の JSON（200 OK / 201 Created）
- **エラーレスポンス**: ステータスコード・エラーコード・説明の一覧 + JSON 例

### エラーレスポンス標準形式

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "[ユーザー向けメッセージ]",
    "details": [
      { "field": "[フィールド名]", "message": "[エラー詳細]" }
    ]
  }
}
```

標準エラーコード一覧:

| ステータス | コード | 説明 |
|-----------|--------|------|
| 400 | VALIDATION_ERROR | バリデーションエラー |
| 401 | UNAUTHORIZED | 認証トークンが無効または未指定 |
| 403 | FORBIDDEN | 操作権限がない |
| 404 | NOT_FOUND | リソースが存在しない |
| 429 | RATE_LIMIT_EXCEEDED | レート制限超過 |

## CRUD パターン別出力例

### 一覧取得（GET /resources）

クエリパラメータ:

| パラメータ | 型 | デフォルト | 説明 |
|-----------|-----|----------|------|
| page | number | 1 | ページ番号 |
| limit | number | 20 | 1ページあたりの件数（最大100） |
| sort | string | createdAt | ソートキー |
| order | string | desc | asc / desc |

レスポンス例（200 OK）:

```json
{
  "data": [
    {
      "id": "uuid-1",
      "name": "サンプルリソース",
      "createdAt": "2026-01-01T00:00:00Z"
    }
  ],
  "pagination": {
    "total": 100,
    "page": 1,
    "limit": 20,
    "totalPages": 5
  }
}
```

空状態レスポンス例:

```json
{
  "data": [],
  "pagination": {
    "total": 0,
    "page": 1,
    "limit": 20,
    "totalPages": 0
  }
}
```

### 新規作成（POST /resources）

リクエストボディ例:

```json
{
  "name": "新しいリソース名",
  "description": "説明（任意）"
}
```

レスポンス例（201 Created）:

```json
{
  "id": "new-uuid",
  "name": "新しいリソース名",
  "description": "説明（任意）",
  "createdAt": "2026-01-01T00:00:00Z"
}
```

バリデーションエラー例（400 Bad Request）:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "入力値に誤りがあります",
    "details": [
      { "field": "name", "message": "name は必須です" }
    ]
  }
}
```

### 詳細取得（GET /resources/:id）

レスポンス例（200 OK）:

```json
{
  "id": "uuid-example",
  "name": "リソース名",
  "description": "説明",
  "createdAt": "2026-01-01T00:00:00Z",
  "updatedAt": "2026-01-01T00:00:00Z"
}
```

### 更新（PATCH /resources/:id）

リクエストボディ例（更新するフィールドのみ）:

```json
{
  "name": "新しいリソース名"
}
```

レスポンス例（200 OK）: 更新後のリソースをそのまま返す

### 削除（DELETE /resources/:id）

レスポンス: `204 No Content`（ボディなし）

論理削除の場合は `200 OK` でステータスフィールドを返すこと。

## 設計品質チェックリスト

エンドポイント生成後に以下を確認する:

| 観点 | チェック内容 |
|------|------------|
| 命名規則 | リソース名が複数形・スネークケースで統一されているか |
| HTTP メソッド | CRUD 操作に適切なメソッドが使用されているか |
| ステータスコード | 操作結果に適切なコードが返されているか |
| エラーレスポンス | 全エンドポイントで統一されたエラー形式か |
| ページネーション | 一覧取得にページネーションが実装されているか |
| 認証 | 必要なエンドポイントに認証が設定されているか |
| バージョニング | API バージョン（/v1/ 等）が考慮されているか |
