# テンプレートカタログ — /aidd-design

利用可能な設計書式の一覧。/aidd-design Step 0 でこのファイルを読み込み、設計対象に最適な書式を選択する。

## カタログ

| 書式 | 用途 | 主要セクション | 使用タイミング |
|------|------|--------------|--------------|
| プロジェクトチャーター | プロジェクトの目的・スコープ・体制・成功基準を定義 | 目的・背景、スコープ、体制・役割、成功基準、リスク | プロジェクト最初期。何を作るか・誰が関わるかを明確にしたい時 |
| 機能仕様書 | システムが提供する機能の一覧と概要を定義 | 機能一覧、機能詳細（入力・処理・出力）、優先度（MUST/WON'T） | Phase/Epic 定義の前段。実装スコープを固めたい時 |
| 技術スタック | 採用する技術・ツールとその選定理由を記録 | 言語・FW・DB・インフラ・CI/CD、選定理由、代替案との比較 | 技術選定時。ADR の前段として整理したい時 |
| アーキテクチャ概要 | システムの全体構成・主要コンポーネントを図と文章で説明 | 全体図、コンポーネント説明、データフロー、外部連携 | アーキテクチャ決定時。チームへの説明資料として |
| NFR 詳細 | 非機能要件（性能・可用性・セキュリティ等）の目標値を定義 | 可用性、性能・拡張性、セキュリティ、運用・保守性 | Epic 定義前。非機能面の制約を明確にしたい時 |
| 画面フロー | 画面遷移・ユーザーフロー・主要画面の構成を定義 | 画面一覧、遷移図、各画面の主要要素 | UI 設計時。モック作成前の情報整理として |
| 外部連携仕様 | 外部 API・サービスとの連携方法を定義 | 連携先一覧、API 仕様（エンドポイント・認証・データ形式）、エラーハンドリング | 外部依存を持つ実装の前段として |

## 書式選択の判断基準

| ユーザーの言葉 | 推奨書式 |
|--------------|---------|
| 「何を作るか決めたい」「プロジェクトを始めたい」 | プロジェクトチャーター |
| 「機能を整理したい」「何を実装するか一覧にしたい」 | 機能仕様書 |
| 「技術選定をしたい」「どのツールを使うか決めたい」 | 技術スタック |
| 「システム構成を整理したい」「全体像を描きたい」 | アーキテクチャ概要 |
| 「性能要件を整理したい」「非機能要件を決めたい」 | NFR 詳細 |
| 「画面設計をしたい」「ユーザーフローを整理したい」 | 画面フロー |
| 「外部 API との連携を整理したい」 | 外部連携仕様 |
