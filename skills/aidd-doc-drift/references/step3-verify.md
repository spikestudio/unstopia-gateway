# Step 3: 乖離確認 — 詳細手順

Step 2 で推論した影響ドキュメント候補を実際に読み、乖離を確認・列挙する。

## 3-1. 候補ドキュメントの精読

Step 2 でユーザーが承認した候補ドキュメントを1件ずつ読む。

確認する乖離の種類:

| 種類 | 例 |
|------|-----|
| **死参照** | 削除・リネームされたファイルへのリンクや言及が残っている |
| **追記漏れ** | 新規追加の内容（スキル・機能・設定等）が一覧・目次・パイプライン図に未記載 |
| **古い記述** | 変更前の名称・仕様・手順が更新されていない |
| **ドキュメント不在** | 変更に対応するドキュメントが存在すべきだが作られていない |

## 3-2. 乖離の記録

確認した乖離を以下の形式で記録する:

```
| # | ドキュメント | 行/箇所 | 乖離の種類 | 内容 | 対応案 |
|---|------------|---------|----------|------|--------|
| 1 | docs/FRAMEWORK.md | L544 横断スキル表 | 追記漏れ | /aidd-doc-drift が未記載 | 1行追加 |
| 2 | CLAUDE.md | §スキルパイプライン | 古い記述 | 旧スキル名 /aidd-xxx が残存 | /aidd-yyy に更新 |
| 3 | — | — | ドキュメント不在 | /aidd-doc-drift の使い方ガイドが存在しない | 新規作成 |
```

**問題が見つからなかった候補:**

```
docs/conventions/naming.md — 乖離なし（変更の影響なし）
```

## 3-3. 乖離なしの場合

全候補を確認して乖離が0件だった場合:

```
確認した全ドキュメントに乖離は見つかりませんでした。
ドキュメントは最新の状態を反映しています。
```

スキルをここで終了し、コミットは不要。

## 3-4. Step 4 への引き渡し

乖離一覧をまとめて Step 4（ブリーフィング・修正実行）に渡す。

乖離の分類を明確にする:

- **更新対象**: 既存ドキュメントを修正するもの
- **新規作成対象**: 存在しないドキュメントを新たに作るもの
