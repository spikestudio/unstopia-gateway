# Step 0-2: 機能意図からのストーリー詳細化 — 詳細手順

## 入力

- Phase 定義書（`docs/requirements/PD-NNN-slug.md`）の「Epic マッピング」セクション
- 対象 Epic に対応する機能意図（1行〜数行の箇条書き）

## 手順

### 0-2-1. 機能意図の取得

Phase 定義書の「Epic マッピング」テーブルから対象 Epic に対応する機能意図を取得する。

```
例: 機能意図 = 「ユーザーが広告枠を登録できる」
```

### 0-2-2. ペルソナの特定

機能意図からペルソナを推定する。複数のペルソナが考えられる場合は全て列挙し、ユーザーに確認する。

```
機能意図「ユーザーが広告枠を登録できる」から以下のペルソナを推定しました:
- 広告主（広告枠を登録したい人）
- 管理者（広告主を管理したい人）

使用するペルソナはこれでよいですか？
```

### 0-2-3. ストーリーの生成

機能意図とペルソナを組み合わせて、「As a [ペルソナ], I want to [行動], so that [理由].」形式のストーリーを生成する。

**生成ルール:**

- 正常系のストーリーを中心に生成する
- 管理系・エラーケース・認可に関するストーリーも補完する
- 1ストーリー = 1つの明確なユーザーアクションに限定する
- 複合条件（「〇〇かつ△△できる」）は分割する

**提示フォーマット:**

```
機能意図「ユーザーが広告枠を登録できる」から以下のストーリーを生成しました:

### 人間の入力（機能意図から生成）
- S1: As a **広告主**, I want to 広告枠の基本情報（名称・形式・サイズ）を登録する, so that 配信先として使用できるようにするため.
- S2: As a **広告主**, I want to 登録した広告枠の内容を確認する, so that 間違いがないか検証できるようにするため.

### AI 補完（抜け漏れ補完）
- S3: As a **広告主**, I want to 広告枠の登録でエラーが発生した場合に原因を確認する, so that 修正して再登録できるようにするため.（AI 補完: エラーハンドリングは UX 上必須）
- S4: As a **管理者**, I want to 広告主が登録した広告枠を一覧で確認する, so that 不正な広告枠を検出・管理できるようにするため.（AI 補完: 管理者視点のストーリーが欠落）

変更・追加したいストーリーがあれば指示してください。
問題なければ「OK」で確定します。
```

### 0-2-4. ストーリーの確認・修正

ユーザーが確認・修正を行う対話フローを実行する。

| 操作 | 例 |
|------|-----|
| 追加 | 「S5: As a ... を追加」 |
| 削除 | 「S3 は不要」 |
| 修正 | 「S1 の so that を変えたい」 |
| 確定 | 「OK」「進めて」 |
