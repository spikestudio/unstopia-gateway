# ストーリードラフトフォーマット（CLI 版）

CLI 仕様から逆算したストーリードラフトの生成ガイド。`/aidd-new-epic` Step 0-2 の入力として使用できる形式で出力する。

## ストーリー生成ルール

> **ストーリー導出ルール**: 詳細は `skills/aidd-inception-mock/references/story-derivation-rules.md` を参照。
> 粒度・ペルソナ推定・AI補完の判定基準・情報源付記は全 Inception 子スキルで共通。

1. **ユーザー視点で記述** — 「`create` コマンドを実行する」ではなく「新しい[リソース]を作成する」と記述する
2. **ペルソナを明示** — CLI の利用者（開発者・運用者・エンドユーザー等）を特定する
3. **コマンドを隠蔽** — ストーリーはビジネス目的で記述し、技術詳細はコマンド参照に留める
4. **根拠を付記** — どのコマンドから導出したかを明記する

## 出力テンプレート

```markdown
## 🖥️ inception-cli セッション成果物

対象機能意図: [機能意図]
CLI フレームワーク: [cobra/click/yargs 等]
コマンド数: N
実施日: [yyyy-mm-dd]

---

## ストーリードラフト

> このセクションは /aidd-new-epic Step 0-2 の入力として使用します。

### 機能意図から導出したストーリー

- S1: As a **[ペルソナ]**, I want to [行動], so that [理由].
  根拠: `[コマンド] [サブコマンド]`

- S2: As a **[ペルソナ]**, I want to [行動], so that [理由].
  根拠: `[コマンド] [サブコマンド] --[フラグ]`

### AI が補完提案したストーリー

- S3: As a **[ペルソナ]**, I want to [行動], so that [理由].
  根拠: [補完理由: シェル補完 / 設定管理 / ヘルプ表示 等]

## /aidd-new-epic への引き渡し方

/aidd-new-epic の Step 0-2 開始時に以下を提示する:

```text
inception-cli セッションの成果物があります。以下を Step 0-2 の入力として使用します:

[上記の成果物をここに貼付]

この内容をもとにストーリーを確認・補完し、AC を定義します。
```

## /aidd-new-epic との対応関係

| inception-cli 成果物 | /aidd-new-epic での使用場所 |
|---------------------|--------------------------|
| ストーリードラフト（機能意図から） | Step 0-2: ストーリー詳細化の入力 |
| ストーリードラフト（AI 補完分） | Step 0-2: 追加ストーリー候補として確認 |
| コマンド一覧 | Step 1: AC 導出時のコマンド参照 |
| フラグ定義 | Step 1: AC の入力条件定義の参考 |
| 使用例 | Step 1: デモシナリオ・AC 確認の参考 |
| 未決定事項 | Epic 仕様書「未決定事項」セクションに転記 |
