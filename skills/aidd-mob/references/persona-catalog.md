# ペルソナカタログ

mob セッションで使用できる専門家ペルソナの一覧。AI がペルソナを自動選択する際の参照カタログ。

## 固定ペルソナ（デフォルト）

設定ファイルで上書きしない限り常時参加するペルソナ。

| ペルソナ | 役割 | 重点視点 |
|---------|------|---------|
| **ビジネスアナリスト** | ビジネス要件・ROI の評価者 | ビジネス価値・ステークホルダー調整・優先度判断 |

## カタログ（AI が選択する追加ペルソナ）

### ビジネス系

| ペルソナ | 選択条件（例） | 重点視点 |
|---------|-------------|---------|
| **プロダクトマネージャー** | 機能の優先度判断が必要な場合 | ユーザーニーズ・競合差別化・ロードマップ |
| **マーケター** | ユーザー獲得・成長施策が絡む場合 | ターゲットユーザー・訴求ポイント・KPI |
| **カスタマーサクセス** | 既存ユーザーへの影響が大きい場合 | オンボーディング・サポートコスト・解約リスク |
| **法務・コンプライアンス** | 個人情報・金融・医療・契約が絡む場合 | 法的リスク・規制準拠・利用規約 |
| **財務アナリスト** | コスト・課金モデルが絡む場合 | 収益モデル・コスト構造・ROI |

### UX・デザイン系

| ペルソナ | 選択条件（例） | 重点視点 |
|---------|-------------|---------|
| **UX デザイナー** | 画面・操作フローが関わる場合 | ユーザビリティ・情報アーキテクチャ・アクセシビリティ |
| **ユーザーリサーチャー** | ユーザー行動の仮説が必要な場合 | ユーザーモデル・ジャーニーマップ・ペインポイント |

### 技術系

| ペルソナ | 選択条件（例） | 重点視点 |
|---------|-------------|---------|
| **バックエンドエンジニア** | API・データ処理・性能が関わる場合 | スケーラビリティ・データモデル・API 設計 |
| **フロントエンドエンジニア** | UI 実装・状態管理が関わる場合 | 実装コスト・ブラウザ制約・レンダリング |
| **インフラ／SRE** | 可用性・デプロイ・監視が関わる場合 | 可用性・コスト・オペレーション負荷 |
| **セキュリティエンジニア** | 認証・認可・個人情報が絡む場合 | 脅威モデル・OWASP・データ保護 |
| **データエンジニア** | 分析・レポート・ML が絡む場合 | データパイプライン・スキーマ設計・整合性 |

### 業界専門家系

| ペルソナ | 選択条件（例） | 重点視点 |
|---------|-------------|---------|
| **広告業界専門家** | 広告配信・媒体・DSP/SSP が関わる場合 | 業界慣行・用語・主要プレイヤー |
| **EC 業界専門家** | 購買フロー・在庫・決済が関わる場合 | 業界標準・ユーザー期待値・競合事例 |
| **医療業界専門家** | 医療情報・患者データが関わる場合 | 法規制・プライバシー・医療倫理 |
| **金融業界専門家** | 決済・与信・資産管理が関わる場合 | 金融規制・リスク管理・業界標準 |
| **SaaS 業界専門家** | B2B SaaS のプロダクト設計全般 | マルチテナント・課金・オンボーディング |

## ペルソナ自動選択の手順

1. 機能意図のキーワードを抽出する（例: 「広告枠登録」→ 広告・登録・管理）
2. カタログの「選択条件」と照合して候補を絞る
3. 固定ペルソナと重複しない候補から最低3名を選択する
4. 各選択に「なぜこのペルソナが有効か」の理由を付記する

## 固定ペルソナの設定方法

固定ペルソナの読み込み優先順位（高い順）:

1. `.aidd-mob-config` ファイル
2. `CLAUDE.md` の `mob_fixed_personas:` セクション
3. デフォルト（ビジネスアナリスト）

### `.aidd-mob-config` の設定例

プロジェクトルートに `.aidd-mob-config` を配置する:

```yaml
# .aidd-mob-config
fixed_personas:
  - name: ビジネスアナリスト
    focus: ビジネス要件・ROI
  - name: セキュリティエンジニア
    focus: セキュリティ・コンプライアンス
min_additional_personas: 3
max_personas: 5
```

### CLAUDE.md での設定例

`CLAUDE.md` に以下のセクションを追記する:

```markdown
## mob_fixed_personas

- name: ビジネスアナリスト
  focus: ビジネス要件・ROI
- name: 法務・コンプライアンス
  focus: 法的リスク・規制準拠
```
