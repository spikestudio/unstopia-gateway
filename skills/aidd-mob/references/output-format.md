## 出力後のアクション

```
🎉 mob セッション完了

成果物を /aidd-new-epic に引き渡す準備ができました。

次のステップ:
→ /aidd-new-epic [Epic名] を実行してください。
→ 上記の成果物を /aidd-new-epic に渡すと、Step 0-2 の AI によるストーリー補完を省略できます。
   ただし、Step 0-2 をスキップするかどうかは /aidd-new-epic 内でユーザーが決定します。
```

## /aidd-new-epic との対応関係

| mob 成果物 | /aidd-new-epic での使用場所 | 詳細 |
|-----------|--------------------------|------|
| ストーリー候補 | Step 0-2: ストーリー詳細化の入力 | `references/step0-story-elaboration.md` の 0-2-3「ストーリーの生成」に渡す。mob 成果物があれば AI 補完を省略できる |
| 制約一覧 | Step 1: AC 導出時の制約条件 | AC 記述時に「← 制約 C1」として参照する |
| 前提条件一覧 | Step 1: AC の前提条件セクション | Epic 仕様書の「バリデーションルール」「エラーケース」に反映 |
| 未決定事項 | Step 1〜3: 設計判断・ADR が必要な箇所の特定 | `/aidd-adr` または Epic 仕様書「未決定事項」に転記 |
| 対立点・トレードオフ | Step 3: 設計・ADR の入力 | `/aidd-adr` で意思決定し ADR を作成する |

### /aidd-inception への引き渡し方（S4: mob→inception 対応表）

mob 成果物を `/aidd-inception` に引き渡す際は以下の対応表に従う:

| mob の成果物 | /aidd-inception での使い方 |
|------------|--------------------------|
| ストーリー候補 | サブスキル起動時のプロンプトに「確定ストーリー候補」として引き渡す |
| 制約一覧 | 各サブスキルの設計時の制約条件として参照 |
| 前提条件一覧 | handoff-format.md の【制約・前提条件】に統合 |
| 未決定事項 | handoff-format.md の【要確認】未決定事項に統合 |
| 対立点・トレードオフ | handoff-format.md の【対立点・トレードオフ】に統合 |
| 参照情報源 | サブスキルのリサーチ参照情報として活用 |

### /aidd-new-epic への引き渡し方

mob セッション成果物を `/aidd-new-epic` に引き渡す際は、Step 0-2 の開始時に以下を提示する:

```
mob セッションの成果物があります。以下を Step 0-2 の入力として使用します:

[mob セッション成果物をここに貼付]

この内容をもとにストーリーを確認・補完します。
```

**引き渡し時の注意:**

- `ストーリー候補` セクションは `/aidd-new-epic` の `step0-story-elaboration.md` 手順 0-2-3「ストーリーの生成」をスキップできる。mob 成果物のストーリー候補を AI 補完なしでそのまま使用し、確認・精緻化のみ行う
- `未決定事項` は Epic 仕様書の「未決定事項」セクションに転記し、AC 定義前に解消する
- `対立点・トレードオフ` は ADR 化が必要なものを `/aidd-adr` に委ねる
