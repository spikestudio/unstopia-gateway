# Step 0: 引数受付・Epic 特定 — 詳細手順

Epic ID の受付・バリデーション、コンテキスト読み込みの詳細手順を定義する。

## 入力

- `$ARGUMENTS`（引数）
- `docs/requirements/` 配下のファイル群
- `docs/domain/`（存在する場合）
- `docs/design/screen-list-*.md`（存在する場合）

## 0-1. Epic の特定

### 引数あり

`/aidd-screen-plan ES-NNN` のように引数が指定された場合:

1. `$ARGUMENTS` から Epic ID（`ES-NNN` 形式）を抽出する
2. `docs/requirements/ES-NNN-*.md` の存在を確認する
3. ファイルが存在しない場合はエラーを表示してスキルを停止する

```
エラー: Epic 仕様書が見つかりません。

指定された Epic ID: ES-NNN
確認したパス: docs/requirements/ES-NNN-*.md

以下を確認してください:
- Epic ID が正しいか（例: ES-001, ES-012）
- Epic 仕様書が作成済みか（未作成の場合は `/aidd-new-epic` を実行）
```

1. ファイルが存在する場合は読み込みを行い、0-2 に進む

### 引数なし

引数が指定されていない場合:

1. `docs/requirements/ES-*.md` を全て列挙する
2. `docs/design/screen-list-*.md` が存在するものを除外する（画面一覧作成済みの Epic はスキップ）
3. 残ったファイルをユーザーに提示して確認を求める

```
対象の Epic が指定されていません。以下の未処理 Epic が見つかりました:

| # | Epic ID | Epic 名 |
|---|---------|--------|
| 1 | ES-NNN | [Epic名] |
| 2 | ES-MMM | [Epic名] |

どの Epic を対象にしますか？（例: `ES-001`）
または `/aidd-screen-plan ES-NNN` のように引数で指定してください。
```

1. ユーザーが選択・入力した Epic ID を使用して処理を続行する

### 推定失敗時

対象 Epic が見つからない場合:

```
対象の Epic を特定できませんでした。以下のいずれかで指定してください:
- `/aidd-screen-plan ES-NNN` — Epic ID を指定して実行
- Epic 仕様書が存在しない場合は `/aidd-new-epic` を先に実行してください
```

## 0-2. バリデーション

Epic ID と Epic 仕様書を特定した後、以下を検証する。

| チェック項目 | 条件 | エラー時の振る舞い |
|------------|------|------------------|
| Epic ID 形式 | `ES-NNN`（N は数字）形式であること | フォーマットエラーを表示し、正しい形式の例を案内 |
| Epic 仕様書の存在 | `docs/requirements/ES-NNN-*.md` が存在すること | エラーメッセージ + `/aidd-new-epic` の実行を案内 |
| AC セクションの有無 | Epic 仕様書に `## ストーリーと受入基準` または `**受入基準:**` が存在すること | 警告を表示し、ドメインモデル起点のみで続行するか確認する |

### AC セクションが空の場合の案内

```
警告: 指定された Epic 仕様書（ES-NNN）に AC が定義されていません。

ドメインモデル起点の画面候補のみで続行することもできますが、
AC 起点の画面を洗い出せないため設計漏れのリスクがあります。

以下のいずれかを選択してください:
1. ドメインモデル起点のみで続行する
2. `/aidd-new-epic ES-NNN` で AC を追加してから再実行する
```

## 0-3. コンテキスト読み込み

| 入力 | 取得元 | 用途 | 任意/必須 |
|------|--------|------|----------|
| Epic 仕様書 | `docs/requirements/ES-NNN-*.md` | AC 一覧・バリデーション・エラーケース | 必須 |
| ドメインモデル | `docs/domain/`（存在する場合） | 集約・エンティティから CRUD 画面を補完 | 任意 |
| 既存画面一覧 | `docs/design/screen-list-*.md`（存在する場合） | 画面 ID 採番の重複回避 | 任意 |

### ドメインモデルの確認

1. `docs/domain/` ディレクトリが存在するか確認する
2. 存在する場合: ファイル一覧を取得し、集約・エンティティ定義を読み込む
3. 存在しない・空の場合: 「ドメインモデルが見つかりませんでした。AC 起点の画面のみで続行します。」と通知する

### 既存画面 ID の確認

1. `docs/design/screen-list-*.md` が存在する場合は読み込む
2. 既存の `SCR-NNN` 形式の最大 ID を記録する（新規採番時の重複防止に使用）
3. 存在しない場合は `SCR-001` から採番を開始する

## 完了時

以下の情報を整理して Step 1 に渡す:

- 対象 Epic ID と Epic 仕様書の内容
- ドメインモデルの有無と内容（存在する場合）
- 既存画面 ID の最大値（採番の起点）
- AC 一覧（Step 1 の解析対象）
