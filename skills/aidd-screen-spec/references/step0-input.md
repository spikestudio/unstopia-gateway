# Step 0: 引数受付・screen-list 読み込み — 詳細手順

`$ARGUMENTS` から `screen-list-*.md` ファイルパスを受け取り、バリデーションを行い、コンテキストを読み込む詳細手順を定義する。

## 入力

- `$ARGUMENTS`（引数）
- `docs/design/screen-list-*.md`（画面一覧ドキュメント）
- `docs/requirements/ES-NNN-*.md`（Epic 仕様書、存在する場合）
- `docs/design/api-spec.md`（API 仕様書、存在する場合）

## 0-1. screen-list の特定

### 引数あり

`/aidd-screen-spec docs/design/screen-list-epic-slug.md` のように引数が指定された場合:

1. `$ARGUMENTS` からファイルパスを抽出する
2. 指定されたファイルの存在を確認する
3. ファイルが存在しない場合はエラーを表示してスキルを停止する

```
エラー: 指定された画面一覧ドキュメントが見つかりません。

指定されたパス: docs/design/screen-list-[slug].md

以下を確認してください:
- ファイルパスが正しいか
- 画面一覧が作成済みか（未作成の場合は `/aidd-screen-plan` を実行）
```

1. ファイルが存在する場合は読み込みを行い、0-2 に進む

### 引数なし

引数が指定されていない場合:

1. `docs/design/screen-list-*.md` を全て列挙する
2. ファイルが複数ある場合はユーザーに選択を求める
3. ファイルが1件のみの場合はそのファイルを使用する旨を通知して確認を求める

```
対象の画面一覧ドキュメントが指定されていません。以下が見つかりました:

| # | ファイル | 更新日時 |
|---|---------|---------|
| 1 | docs/design/screen-list-[slug].md | [日時] |
| 2 | docs/design/screen-list-[slug2].md | [日時] |

どの画面一覧を対象にしますか？（例: `1` または `docs/design/screen-list-[slug].md`）
```

1. ユーザーが選択したファイルを使用して処理を続行する

### ファイルが存在しない場合

`docs/design/screen-list-*.md` が1件も存在しない場合:

```
エラー: 画面一覧ドキュメントが見つかりません。

確認したパス: docs/design/screen-list-*.md

画面一覧ドキュメントを作成するには `/aidd-screen-plan` を先に実行してください。
```

スキルを停止する。

## 0-2. バリデーション

screen-list ファイルを特定した後、以下を検証する。

| チェック項目 | 条件 | エラー時の振る舞い |
|------------|------|------------------|
| ファイルの存在 | 指定された `screen-list-*.md` が存在すること | エラーメッセージ + `/aidd-screen-plan` の実行を案内してスキル停止 |
| SCR-ID の存在 | screen-list に `SCR-NNN` 形式の ID が1件以上あること | 警告を表示し、続行するか確認する |
| SCR-ID の形式 | `SCR-NNN`（3桁ゼロ埋め）形式であること | 不正な ID を警告表示し、screen-list の確認を促す |

### SCR-ID が0件の場合

```
警告: 指定された画面一覧ドキュメント（[ファイル名]）に SCR-ID が見つかりません。

画面が0件の状態では仕様書を生成できません。
以下のいずれかを選択してください:
1. `/aidd-screen-plan` で画面一覧を作成・更新する
2. 別の画面一覧ファイルを指定して再実行する
```

続行しない（スキルを停止する）。

### SCR-ID の形式が不正な場合

```
警告: 以下の SCR-ID の形式が正しくない可能性があります:
- [不正な ID]（期待形式: SCR-NNN、例: SCR-001）

screen-list ファイルを確認してください: [ファイルパス]
```

警告のみ表示し、正しい形式の SCR-ID が1件以上あれば続行する。

## 0-3. コンテキスト読み込み

| 入力 | 取得元 | 用途 | 任意/必須 |
|------|--------|------|----------|
| 画面一覧ドキュメント | `docs/design/screen-list-[slug].md` | SCR-ID 一覧・画面名・種別・AC トレース | 必須 |
| Epic 仕様書 | `docs/requirements/ES-NNN-*.md` | AC 詳細・バリデーションルール・ロール定義 | 任意 |
| API 仕様書 | `docs/design/api-spec.md`（存在する場合） | API エンドポイント参照 | 任意 |
| 既存画面仕様書 | `docs/design/screen-spec-*/`（存在する場合） | 重複回避・構造参照 | 任意 |

### 画面一覧ドキュメントの読み込み

1. ファイルを読み込み、全 SCR-ID と画面名を抽出する
2. 各 SCR-ID に対応する画面種別（一覧・詳細・フォーム等）を記録する
3. AC トレース（`← AC-ENNN-NN`）が記載されている場合は記録する

### Epic 仕様書の確認

1. `docs/requirements/ES-*.md` を列挙し、screen-list の出所となる Epic を特定する（screen-list ファイル名の slug から推定、または画面一覧内に記載の Epic ID を参照）
2. 特定できた場合は Epic 仕様書を読み込み、AC・バリデーションルール・ロール定義・エラーケースを取得する
3. 特定できない場合は「Epic 仕様書が特定できませんでした。AC なしで続行します。」と通知して続行する

### API 仕様書の確認

1. `docs/design/api-spec.md` が存在するか確認する
2. 存在する場合: 読み込んで API エンドポイント一覧を取得する（Step 1 での参照に使用）
3. 存在しない場合: 「API 仕様書が見つかりませんでした。API エンドポイント参照は「TBD」として記録します。」と通知する

### 既存画面仕様書の確認

1. `docs/design/screen-spec-*/` が存在するか確認する
2. 存在する場合: 作成済み SCR-ID を記録し、再生成を避けるか確認する
3. 存在しない場合: Step 1 で新規生成を開始する

## 完了時

以下の情報を整理して Step 1 に渡す:

- 対象 screen-list ファイルパスと全 SCR-ID 一覧
- Epic 仕様書の内容（特定できた場合）
- API 仕様書の内容（存在する場合）
- 既存画面仕様書の SCR-ID 一覧（存在する場合）
- 処理対象 SCR-ID の総数（進捗管理に使用）
