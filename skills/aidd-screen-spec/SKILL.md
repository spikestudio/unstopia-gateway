# /aidd-screen-spec — 画面仕様書・画面遷移図生成

`/aidd-screen-plan` が生成した画面一覧ドキュメント（`docs/design/screen-list-*.md`）を入力として、画面単位の仕様書（フィールド定義・バリデーション・状態設計・ロール・API参照）と画面遷移図（Mermaid形式・トリガー条件・条件付き遷移）を生成するスキル。

> **このスキルの目的は「画面一覧から実装者が設計を解釈せずに開発を進められる標準フォーマットの仕様書を生成し、後続スキル `/aidd-screen-ui` に接続できる成果物を出力する」こと。**

## インターフェース

### 前提条件

| 前提 | 確認方法 | 未完了時の案内 |
|------|---------|-------------|
| 画面一覧ドキュメントが作成済み | `docs/design/screen-list-*.md` の存在確認 | `/aidd-screen-plan` を先に実行 |

### 入力

| 項目 | 形式 | 必須 | 説明 |
|------|------|------|------|
| screen-list ファイルパス | 引数 | 任意 | 指定なしなら未処理の画面一覧を自動推定 |

### capability 呼び出し

| capability | Step | 引数 |
|-----------|------|------|
| /aidd-research | Step 1 | ドメイン知識の補完（必要時のみ） |

### 出力

| 項目 | 形式 | 説明 |
|------|------|------|
| 画面仕様書 | Markdown | `docs/design/screen-spec-[slug]/SCR-NNN-[name].md`（画面単位） |
| 画面遷移図 | Markdown | `docs/design/screen-flow-[slug].md`（Mermaid形式） |

### 後続スキル

| スキル | 条件 |
|--------|------|
| `/aidd-screen-ui` | 画面仕様書承認後、UI 実装に進む場合 |
| `/aidd-next` | G3 マイルストーン判定 |

## Step 0: 引数受付・screen-list 読み込み

`$ARGUMENTS` から screen-list ファイルパスを取得する。指定がなければ `docs/design/screen-list-*.md` を列挙してユーザーに選択を求める。ファイルが存在しない場合はエラーを表示して停止する。

### 0-1. screen-list の特定

- **引数あり**（例: `/aidd-screen-spec docs/design/screen-list-epic-slug.md`）: 指定ファイルの存在を確認して読み込む。存在しない場合はエラーを表示して停止する
- **引数なし**: `docs/design/screen-list-*.md` を全件列挙してユーザーに選択を求める。1件のみの場合はそのファイルを使用する旨を通知して確認を求める
- **ファイルが存在しない場合**（引数なし・0件）:

```
エラー: 画面一覧ドキュメントが見つかりません。

確認したパス: docs/design/screen-list-*.md

画面一覧ドキュメントを作成するには `/aidd-screen-plan` を先に実行してください。
```

### 0-2. バリデーション

| チェック項目 | 条件 | エラー時の振る舞い |
|------------|------|------------------|
| ファイルの存在 | 指定された `screen-list-*.md` が存在すること | エラーメッセージ + `/aidd-screen-plan` の実行を案内してスキル停止 |
| SCR-ID の存在 | screen-list に `SCR-NNN` 形式の ID が1件以上あること | 警告を表示し、続行するか確認する（0件の場合はスキル停止） |
| SCR-ID の形式 | `SCR-NNN`（3桁ゼロ埋め）形式であること | 不正な ID を警告表示し、正しい形式の SCR-ID が1件以上あれば続行 |

### 0-3. コンテキスト読み込み

| 入力 | 取得元 | 用途 | 任意/必須 |
|------|--------|------|----------|
| 画面一覧ドキュメント | `docs/design/screen-list-[slug].md` | SCR-ID 一覧・画面名・種別・AC トレース | 必須 |
| Epic 仕様書 | `docs/requirements/ES-NNN-*.md` | AC 詳細・バリデーション・ロール定義 | 任意 |
| API 仕様書 | `docs/design/api-spec.md`（存在する場合） | API エンドポイント参照 | 任意 |
| 既存画面仕様書 | `docs/design/screen-spec-*/`（存在する場合） | 重複回避・構造参照 | 任意 |

詳細手順は `references/step0-input.md` を参照して実行する。

## Step 1: 画面仕様書生成

画面一覧の各 SCR-ID に対して画面仕様書を生成する。フィールド定義・バリデーション（Step 1a）と状態設計・ロール・API参照（Step 1b）の2フェーズで進める。

各画面の仕様書生成後にユーザーの確認を得てから次の画面へ進む。

**生成内容:**

- フィールド定義（フィールド名・型・必須/任意）— フィールドがない場合は「フィールドなし」と明記
- バリデーションルール（形式・文字数・依存条件）— 未定義の場合は「TBD」と明記
- 状態設計（空状態・ローディング状態・エラー状態）
- ロール定義（閲覧可能なロール・操作可能なロール）— 未定義の場合は「全ロール」または「要確認」と記録
- API エンドポイント参照（パス・HTTPメソッド）— 未確定の場合は「TBD」と記録

### Step 1a: フィールド定義・バリデーション生成

各 SCR-ID に対してフィールド定義とバリデーションルールを生成する対話フロー。

**生成内容:**

- フィールド定義（フィールド名・型・必須/任意）— フィールドがない場合は「フィールドなし」と明記（AC-E053-01, AC-E053-02）
- バリデーションルール（形式・文字数・依存条件）— 未定義の場合は「TBD」と明記し続行（AC-E053-03, AC-E053-04）

詳細手順は `references/step1a-fields.md` を参照して実行する。

### Step 1b: 状態設計・ロール・API参照生成

各 SCR-ID に対して状態設計・ロール定義・API エンドポイント参照を生成する対話フロー。

**生成内容:**

- 状態設計（空状態・ローディング状態・エラー状態）（AC-E053-05, AC-E053-06）
- ロール定義（閲覧可能なロール・操作可能なロール）— 未定義の場合は「全ロール」または「要確認」と記録（AC-E053-07, AC-E053-08）
- API エンドポイント参照（パス・HTTPメソッド）— 未確定の場合は「TBD」と記録（AC-E053-09, AC-E053-10）

詳細手順は `references/step1b-state-role-api.md` を参照して実行する。

## Step 2: 画面遷移図生成

確定した画面仕様書の全 SCR-ID を基に Mermaid 形式の画面遷移図を生成する。トリガー条件・条件付き遷移を含む。

**生成内容:**

- 全 SCR-ID を含む Mermaid 形式の遷移図（AC-E053-11）
- 各矢印にトリガー条件ラベル（成功・エラー・キャンセル・タイムアウト）（AC-E053-12）
- 各遷移に4種のトリガー条件を対話的に定義（AC-E053-13）— トリガーが1種のみの遷移は他を省略可（AC-E053-14）
- ユーザーロールやデータ状態による条件付き遷移（条件ラベル付き）（AC-E053-15, AC-E053-16）

ユーザーの明示的な承認を得てから Step 3 に進む。

詳細手順は `references/step2-flow.md` を参照して実行する。

## Step 3: ブリーフィング・完了

`references/briefing-spec.md` に従ってブリーフィングを実行する。

**重点項目:**

- 生成した画面仕様書の数と AC カバレッジ結果
- 未定義（TBD・要確認）の項目一覧
- 画面遷移図のカバレッジ（全 SCR-ID が含まれているか）
- 後続スキル `/aidd-screen-ui` への接続フォーマットの確認

ユーザーの承認後、完了メッセージを表示する。

```
## 画面仕様書・遷移図生成完了

**入力:** docs/design/screen-list-[slug].md
**出力ディレクトリ:** docs/design/screen-spec-[slug]/
**画面遷移図:** docs/design/screen-flow-[slug].md
**生成した仕様書:** N 件
**TBD 項目:** N 件（バリデーション: N / API: N）

### 次のステップ
- `/aidd-screen-ui` で画面 UI の実装に進む
- `/aidd-next` で G3 マイルストーン判定を進める
```

詳細手順は `references/briefing-spec.md` を参照して実行する。

## 重要なルール

- **日本語で対話する** — フレームワーク全体の対話言語が日本語であるため
- **FRAMEWORK.md の意思決定基準に従う**
- **各ステップでユーザーの確認を得てから次に進む** — 自動的に全ステップを実行しない。ユーザーの意図と成果物のずれを早期に検出するため
- **ユーザーの明示的な承認なしに次ステップへ遷移しない** — 「OK」「進めて」「承認」等の明確な意思表示が必要
- **未定義項目でエラー停止しない** — バリデーションルール未定義は「TBD」、ロール未定義は「全ロール」または「要確認」、API 未確定は「TBD」として続行する
- **フィールドなし画面を適切に処理する** — フィールドを持たない画面（ナビゲーションのみ等）では「フィールドなし」と明記する
- **AC トレーサビリティを維持する** — 各仕様書に導出元 AC-ID（`AC-ENNN-NN`）を記録する
- **ブリーフィング省略禁止** — Step 3 完了時に `references/briefing-spec.md` に従ったブリーフィングを必ず実行する
- **各ステップの詳細手順は `references/` を読んで実行する** — SKILL.md 内に手順を複製しない
