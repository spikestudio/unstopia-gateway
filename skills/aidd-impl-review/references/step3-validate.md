# Step 3: バリデーション + フィルタリング — 詳細手順

Step 2 で検出された各問題に対して、専用バリデーションサブエージェントを並列起動し、HIGH SIGNAL か否かを再確認する。

## 入力

- Step 2 で各エージェントが返した問題一覧
- PR diff + CLAUDE.md/conventions + PR タイトル・本文

## バリデーションサブエージェントの起動

Step 2 の各問題について、問題の種類に応じたモデルで **並列** にバリデーションサブエージェントを起動する:

| 問題の種類 | バリデーションモデル | 確認内容 |
|----------|-----------------|---------|
| バグ・ロジック問題 | **Opus** | この問題が実際に発生するか。false positive でないか |
| CLAUDE.md 違反 | **Sonnet** | 引用したルールがこのファイルのスコープに本当に適用されるか。実際に違反しているか |
| conventions 違反 | **Sonnet** | 引用したルールが実際に違反されているか |

各バリデーションサブエージェントへの指示に含める情報:

- PR タイトル・本文（変更の意図）
- 問題の説明と該当コード
- 関連する CLAUDE.md/conventions のルール
- PR diff（コンテキスト確認用）

## フィルタリング

バリデーションサブエージェントが「HIGH SIGNAL」と判断した問題のみを最終リストに含める。

**HIGH SIGNAL の判定基準（サブエージェントへ伝達）:**

- この問題は実際に発生する（仮定・推測ではない）
- PR の変更で導入された問題である（pre-existing でない）
- linter・型チェッカーが自動で検出する問題ではない
- CLAUDE.md/conventions のルールが明確にこのケースを対象にしている

**バリデーション結果が「false positive」の場合:** 最終リストから除外する。

## 問題が 0 件の場合

全問題が false positive と判定された場合も Step 4 へ進む（「問題なし」として出力）。

## 成果物

HIGH SIGNAL 確認済みの問題最終リスト（0 件も含む）
