# Step 4: 出力処理 — 詳細手順

Step 3 のバリデーション済み問題リストを基に、端末出力および GitHub コメント投稿を行う。

## 端末出力（常に実行）

`--comment` の有無に関わらず、常に端末に結果を日本語で出力する。

**問題ありの場合:**

```
## コードレビュー結果: <PR番号> [PR タイトル]

N 件の問題が検出されました:

1. [問題の説明（日本語）]
   観点: [CLAUDE.md準拠/conventions/バグ/セキュリティ]
   場所: <ファイルパス>:<行番号>

2. ...
```

**問題なしの場合:**

```
## コードレビュー結果: <PR番号> [PR タイトル]

問題なし。バグと CLAUDE.md/規約への準拠を確認しました。
```

## GitHub インラインコメント投稿（`--comment` 指定時のみ）

### 問題ありの場合

各問題に対して `gh api repos/.../pulls/.../comments` または `mcp__github_inline_comment__create_inline_comment` でインラインコメントを投稿する。

**コメント形式:**

```
[問題の説明（日本語）]

関連するルール: [CLAUDE.md/conventions の該当ルールへのリンク]（該当する場合）
```

**コミット可能な提案ブロック（条件付き）:**

以下の条件を**すべて**満たす場合のみ、コミット可能な提案ブロックを付与する:

- 修正が 5 行以内で収まる
- 修正が単一箇所で完結する（複数箇所の変更が不要）
- 提案の内容がそのままコミットして問題を完全に解消する

```suggestion
[修正後のコード]
```

条件を満たさない場合は説明のみ（提案ブロックなし）。

**重要: 1 つの問題に対してコメントは 1 件のみ。** 重複投稿禁止。

### 問題なしの場合

```bash
gh pr comment <PR番号> --body "## コードレビュー

問題なし。バグと CLAUDE.md/規約への準拠を確認しました。"
```

## 言語ルール

- 問題の説明・コメント本文: **日本語**
- コードの引用・ファイルリンク・コード提案ブロック: **英語（そのまま）**
- リンク形式（インラインコメント時）:
  `https://github.com/<owner>/<repo>/blob/<full-sha>/<path>#L<start>-L<end>`
  （full SHA は `git rev-parse HEAD` で取得。ブランチ名ではなく SHA を使用すること）
