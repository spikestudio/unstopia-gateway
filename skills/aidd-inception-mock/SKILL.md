# /aidd-inception-mock — 画面モック + ストーリードラフト生成

G1 後・`/aidd-new-epic` 実行前に任意で使用するスキル。機能意図を受け取り、AI が対話的にリサーチ・補完を行いながら画面モックと主要ストーリードラフトを共同制作する。G3 は省略しない（探索的モックは参考資料）。

## インターフェース

### 前提条件

| 前提 | 確認方法 | 未完了時の案内 |
|------|---------|-------------|
| G1 通過済み（Phase 定義書が承認済み） | `docs/requirements/PD-*.md` にステータス「承認済み」が存在する | `/aidd-new-phase` を先に実行 |

### capability 呼び出し

| capability | Step | 引数 |
|-----------|------|------|
| `/aidd-research` | Step 1 | 機能意図に関連する類似サービス・UI パターン・業界ベストプラクティス |

### 入力

| 項目 | 形式 | 必須 | 説明 |
|------|------|------|------|
| 機能意図 または 機能名 | 引数 | 任意 | 省略時はユーザーに確認 |
| 参考資料 | 会話内 | 任意 | 類似サービス URL・既存デザイン等 |

### 出力

| 項目 | 形式 | 説明 |
|------|------|------|
| 画面一覧 | Markdown テーブル | 画面名・種別・主要操作・対応する機能意図 |
| 画面モック | Markdown | テキストベースのローファイモック（全主要画面） |
| ストーリードラフト | Markdown | 画面から逆算した主要ストーリー一覧 |
| 引き渡しメモ | Markdown | `/aidd-new-epic` Step 0 インプット互換フォーマット |

### 後続スキル

| スキル | 条件 |
|--------|------|
| `/aidd-new-epic` | モック・ストーリードラフト完成後、成果物を引き渡して Epic 定義に進む |

## Step 0: 入力受付

1. 機能意図が未指定の場合はユーザーに確認する:

   ```
   /aidd-inception-mock を開始しますか？
   - 対象機能意図または機能名を入力してください
   ```

2. Phase 定義書から対象機能意図を取得し、セッション開始を宣言する:

   ```
   🎨 inception-mock セッション開始
   対象機能意図: [機能意図]

   まず類似サービスをリサーチします...
   ```

## Step 1: 入力収集・AI リサーチ

**AC-E065-01: リサーチと見落とし補完:**

1. `/aidd-research` を呼び出して以下を調査する:
   - 類似サービスの画面構成・UI パターン
   - 業界ベストプラクティス（UX・ナビゲーション・フォームデザイン等）
   - 対象ドメイン特有の規制・制約

2. リサーチ結果を以下の形式でまとめて提示し、**Step 2 での画面候補生成に使用する**（AC-E070-03）:

   ```
   📚 リサーチ結果サマリー

   類似サービスの傾向: [主要な UI パターン]
   業界ベストプラクティス: [参考にすべき設計原則]
   ドメイン固有の注意点: [法的・規制上の制約等]
   補完候補（Step 2 で提案する画面）: [エラー画面・空状態・権限制御等]

   情報源: [URL または文書名]
   ```

   > このサマリーは Step 2 の画面候補洗い出しに直接使用する。
   > 特に「補完候補」セクションを Step 2 の AI 補完提案の根拠とする。

**AC-E065-02: 見落としがちな画面の補完提案:**

リサーチ結果をもとに、以下のカテゴリの画面候補を自動提案する:

- エラー画面（入力エラー・権限エラー・404 等）
- 空状態（データが0件の状態）
- 権限制御（管理者専用画面・閲覧専用モード等）
- 確認・完了画面（送信確認・処理完了等）

参考資料（URL・デザイン等）がある場合は入力を促す。

## Step 2: 画面候補の洗い出し・対話確定

**AC-E065-03: 画面候補の承認フロー:**

1. 機能意図 + リサーチ結果から画面候補を列挙する:

   ```
   📋 画面候補一覧（N 画面）

   | # | 画面名 | 種別 | 主要操作 | 対応する機能意図 |
   |---|--------|------|---------|----------------|
   | 1 | [画面名] | [一覧/入力/詳細/etc.] | [主な操作] | [機能意図] |
   ...

   AIが補完提案した画面:
   - [エラー画面]: [理由]
   - [空状態画面]: [理由]
   ...

   この構成でよいですか？追加・削除があればお知らせください。
   ```

2. ユーザーが承認するまで次ステップに進まない
3. 追加・削除の指示を反映して再提示する
4. **0件になった場合の処理:** リサーチ結果・機能意図から補完候補を自動生成して再提示する。補完候補も断られた場合はセッションを終了する:

   ```
   画面候補が0件になりました。以下の補完候補を提案します:
   [機能意図・リサーチから導出した補完候補]
   
   これも不要な場合はセッションを終了します。
   ```

## Step 3: ローファイモック生成

詳細テンプレートは `references/mock-template.md` を参照して実行する。

**AC-E065-04: テキストモック生成:**

確定した全画面のテキストベースのローファイモックを生成する。
フォーマットは `references/mock-template.md` に従うこと。
各モックに「ユーザーはここで何をするか」のコメントを付記する。

**AC-E065-05: 1画面ずつ確認・修正:**

```
🖼️ [画面名] のモック（N/M）

[モック内容]

---
この画面でよいですか？（OK / 修正指示を入力）
```

ユーザーの指示を反映してモックを更新し、次の画面に進む。

**AC-E065-09: UX フィードバック:**

全モック確認後、UX 観点でのフィードバックを提示する:

```
💡 UX フィードバック

- ナビゲーション: [一貫性の確認]
- 入力フィードバック: [バリデーション・エラー表示の確認]
- エラー表示: [エラーケースのカバレッジ確認]
- 改善提案: [具体的な提案]
```

## Step 4: ストーリードラフト導出

詳細フォーマットは `references/story-draft-format.md` を参照して実行する。

**AC-E065-06: ストーリー逆算:**

確定した画面・操作からストーリーを逆算する。

```
📝 ストーリードラフト

人間が定義した機能意図から:
- S1: As a [ペルソナ], I want to [行動], so that [理由].
  根拠: [対応画面名]

AI が補完提案したストーリー:
- S2: As a [ペルソナ], I want to [行動], so that [理由].
  根拠: [モックのどの操作から導出したか]
```

## Step 5: 引き渡し準備

詳細フォーマットは `references/handoff-format.md` を参照して実行する。

**AC-E065-07: /aidd-new-epic 互換フォーマットで出力:**

画面モック・ストーリードラフト・未決定事項を `/aidd-new-epic` Step 0 インプット互換フォーマットで出力する。

出力後:

```
🎉 inception-mock セッション完了

成果物を /aidd-new-epic に引き渡す準備ができました。

次のステップ:
→ /aidd-new-epic [Epic名] を実行してください。
→ Step 0（ストーリー詳細化）で上記の成果物を参照します。
→ G3（/aidd-screen-plan〜/aidd-screen-ui）は G2 後に通常通り実行します。
   ※ Inception モックは G3 の参考資料として活用してください。
```

## 重要なルール

- **発散フェーズ**: このスキルは発散フェーズに属する。アイデア・ストーリー候補・制約・対立点を積極的に増やすことが目的。成果物は「叩き台」であり、`/aidd-new-epic` の収束フェーズで精緻化される
- 日本語で対話する
- **FRAMEWORK.md の意思決定基準に従う**
- **ユーザー確認なしの自動実行禁止** — 画面候補の承認、各モックの確認は必ずユーザーに委ねる
- **G3 省略を提案しない** — Inception モックは探索的成果物であり G3 を代替しない
- **テキストモックのみ使用** — ASCII アートは使用しない。Markdown の構造で表現する
- ブリーフィング省略禁止
